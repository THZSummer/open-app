# 前端开发规范文档

## 一、项目结构规范

### 1.1 页面文件夹结构

```
每个页面都需要单独用一个文件夹，除非是新增、编辑、详情页可以通过URL参数区分，其余页面必须一个页面一个文件夹。

每个页面文件夹结构如下：
page-name/
├── index.js              # 页面入口文件（主组件）
├── route.js              # 页面路由配置文件
├── thunk.js              # 页面API接口调用文件
├── constant.js           # 页面常量、配置、表格列等文件
├── index.module.less     # 页面样式文件
└── components/           # 页面组件文件夹
    ├── ComponentName.js           # 组件逻辑文件
    └── ComponentName.module.less  # 组件样式文件
```

**文件说明：**

| 文件名               | 说明                               |
| ----------------- | -------------------------------- |
| index.js          | 作为页面的入口文件，定义主组件                  |
| route.js          | 页面的路由配置文件                        |
| thunk.js          | 页面调用哪些接口文件                       |
| constant.js       | 页面的常量、表格table配置、表单校验方法等固定数据或配置内容 |
| index.module.less | 存放页面样式，index.js和组件文件不要写行内样式      |
| components/       | 存放页面组件文件，组件文件名称为组件名称，需有对应的样式文件   |

**命名规范：**

- 推荐使用 `.js` 和 `.module.less` 作为后缀
- 组件文件名称与组件名称保持一致
- 样式文件名称为组件文件名 + `.module.less`

### 1.2 公共目录结构

```
src/
├── components/           # 公共组件目录（被3个及以上页面共享的组件）
├── configs/              # 配置文件目录
│   └── web.config.js     # API接口配置
├── utils/                # 工具函数目录
│   ├── common.js         # 通用工具函数
│   ├── constant.js       # 全局常量
│   └── webFetch.js       # fetch请求封装
└── router/               # 路由目录
```

## 二、文件编写规范

### 2.1 index.js 页面入口文件

#### 2.1.1 代码结构规范

**入口文件内容尽可能抽取为组件**，放到页面的 components 或者公共的 components 组件中。

index.js 尽可能只做以下工作：

1. **页面权限判断**：检查用户是否有权限进入该页面
2. **定义通用数据和方法**：会在 index 中的多个组件内使用的数据或方法
3. **组合页面组件**：将抽取的组件组合在一起

#### 2.1.2 代码重复度控制

**生成代码时，必须满足以下要求：**

1. 查找当前所有项目中的其他页面或组件文件
2. 代码行重复度不能超过10行
3. 如有重复，使用以下方法减少重复度：
   - 修改参数名称
   - 修改对象字段位置
   - 修改文件名称
   - 抽取公共方法到 utils/common.js

#### 2.1.3 代码结构示例

```javascript
/**
 * ========================================
 * 页面名称 - 页面描述
 * ========================================
 *
 * 功能：
 * - 功能点1
 * - 功能点2
 */

import React, { useState, useEffect } from 'react';
import { message, Table, Pagination } from 'antd';
import {
  getList,
  deleteItem
} from './thunk';
import {
  getTableColumns
} from './constant';
import { DEFAULT_PAGINATION, PAGE_SIZE_OPTIONS } from '../../../utils/constant';
import less from './index.module.less';
import SearchBar from './components/SearchBar';
import TableList from './components/TableList';
import FormModal from './components/FormModal';
import ConfirmModal from '../../../components/ConfirmModal/ConfirmModal';

/**
 * 页面主组件
 */
const PageName = () => {
  // ========== State定义 ==========

  const [dataSource, setDataSource] = useState([]);
  const [loading, setLoading] = useState(false);
  const [pagination, setPagination] = useState(DEFAULT_PAGINATION);
  const [queryParams, setQueryParams] = useState(DEFAULT_PAGINATION);

  // 弹窗相关
  const [modalVisible, setModalVisible] = useState(false);
  const [modalTitle, setModalTitle] = useState('新增');
  const [editingId, setEditingId] = useState(null);
  const [submitting, setSubmitting] = useState(false);

  // ========== 通用数据和方法 ==========

  /**
   * 获取列表数据
   */
  const fetchData = async () => {
    setLoading(true);
    const result = await getList(queryParams);
    if (result && result.code === '200') {
      const responseData = result.data || {};
      setDataSource(responseData.list || []);
      setPagination({
        pageNum: Number(responseData.pageNum) || 1,
        pageSize: Number(responseData.pageSize) || DEFAULT_PAGINATION.pageSize,
        total: Number(responseData.total) || 0
      });
    } else {
      message.error(result?.messageZh || result?.message || '获取数据失败');
      setDataSource([]);
    }
    setLoading(false);
  };

  useEffect(() => {
    fetchData();
  }, [queryParams]);

  // ========== 通用回调 ==========

  const handleRefresh = () => {
    fetchData();
  };

  const handlePageChange = (page, pageSize) => {
    setQueryParams({
      ...queryParams,
      pageNum: page,
      pageSize: pageSize || queryParams.pageSize
    });
  };

  const handleEdit = (record) => {
    setModalTitle('编辑');
    setEditingId(record.id);
    setModalVisible(true);
  };

  const handleModalClose = () => {
    setModalVisible(false);
    setEditingId(null);
  };

  // ========== JSX渲染 ==========

  return (
    <div className={less.container}>
      <SearchBar
        queryParams={queryParams}
        onQueryParamsChange={setQueryParams}
      />

      <TableList
        dataSource={dataSource}
        loading={loading}
        pagination={pagination}
        onEdit={handleEdit}
        onRefresh={handleRefresh}
      />

      <Pagination
        total={pagination.total}
        current={pagination.pageNum}
        pageSize={pagination.pageSize}
        pageSizeOptions={PAGE_SIZE_OPTIONS}
        showSizeChanger
        showQuickJumper
        showTotal={(total) => `共 ${total} 条`}
        onChange={handlePageChange}
      />

      <FormModal
        visible={modalVisible}
        title={modalTitle}
        editingId={editingId}
        onClose={handleModalClose}
        onRefresh={handleRefresh}
      />
    </div>
  );
};

export default PageName;
```

### 2.3 thunk.js API接口文件

#### 2.3.1 标准格式

**thunk中所有方法都必须使用try-catch包裹fetchApi。**

```javascript
/**
 * 模块名称 API接口
 * 提供XXX的增删改查功能
 */
import API_CONFIG from '../../../configs/web.config';
import { buildApiUrl, fetchApi } from '../../../utils/webFetch';

/**
 * 获取列表
 * @param {Object} params - 查询参数
 * @returns {Promise<Object>} 列表数据
 */
export const getList = async (params = {}) => {
  try {
    const result = await fetchApi(API_CONFIG.LIST, { method: 'GET', params });
    return result || {};
  } catch (err) {
    return {};
  }
};

/**
 * 获取详情
 * @param {string} id - ID
 * @returns {Promise<Object>} 详情数据
 */
export const getDetail = async (id) => {
  try {
    const result = await fetchApi(buildApiUrl(API_CONFIG.DETAIL, { id }));
    return result || {};
  } catch (err) {
    return {};
  }
};

/**
 * 新增
 * @param {Object} data - 表单数据
 * @returns {Promise<Object>} 新增结果
 */
export const create = async (data) => {
  try {
    const result = await fetchApi(API_CONFIG.CREATE, {
      method: 'POST',
      body: JSON.stringify(data)
    });
    return result || {};
  } catch (err) {
    return {};
  }
};

/**
 * 编辑
 * @param {string} id - ID
 * @param {Object} data - 表单数据
 * @returns {Promise<Object>} 编辑结果
 */
export const update = async (id, data) => {
  try {
    const result = await fetchApi(buildApiUrl(API_CONFIG.UPDATE, { id }), {
      method: 'PUT',
      body: JSON.stringify(data)
    });
    return result || {};
  } catch (err) {
    return {};
  }
};

/**
 * 删除
 * @param {string} id - ID
 * @returns {Promise<Object>} 删除结果
 */
export const delete = async (id) => {
  try {
    const result = await fetchApi(buildApiUrl(API_CONFIG.DELETE, { id }), { method: 'DELETE' });
    return result || {};
  } catch (err) {
    return {};
  }
};
```

#### 2.3.2 规范要求

1. **必须使用 try-catch 包裹**：thunk中所有方法都必须使用try-catch包裹fetchApi
2. **正常返回**：`return result || {}`
3. **异常返回**：`return {}`
4. **添加注释**：每个接口函数必须添加 JSDoc 注释

#### 2.3.3 try-catch 使用场景说明

**thunk中的try-catch**：必须包裹fetchApi，用于捕获网络请求异常

**其他地方（如index.js、组件）调用thunk方法时**：

| 场景 | 是否需要try-catch | 说明 |
|------|-------------------|------|
| 直接使用result | 不需要 | thunk已处理异常 |
| 对result.data做JSON.parse | 需要 | 可能抛出异常 |
| 对result做其他数据处理 | 根据情况 | 可能抛出异常时需要 |

**示例：index.js中调用thunk方法**

```javascript
// 场景1：直接使用result，不需要额外try-catch
const handleSearch = async (params) => {
  setLoading(true);
  const result = await getList(params);
  if (result && result.code === '200') {
    setData(result.data || []);
  } else {
    message.error(result?.messageZh || result?.message || '获取数据失败');
  }
  setLoading(false);
};

// 场景2：需要对result做数据处理时添加try-catch
const handleLoadDetail = async (id) => {
  setLoading(true);
  const result = await getDetail(id);
  if (result && result.code === '200') {
    try {
      const configObj = JSON.parse(result.data.orchestrationConfig);
      setConfig(configObj);
    } catch (parseError) {
      message.error('加载流程配置失败：配置数据格式错误');
    }
  } else {
    message.error(result?.messageZh || result?.message || '加载详情失败');
  }
  setLoading(false);
};
```

#### 3.2.3 webFetch.js

```javascript
/**
 * 动态替换 URL 中的占位符
 * @param {string} template - URL 模板
 * @param {Object} params - 替换参数对象
 * @returns {string} 替换后的 URL
 */
export const buildApiUrl = (template, params = {}) => {
  let url = template;
  Object.keys(params).forEach((key) => {
    url = url.replace(`{${key}}`, params[key]);
  });
  return url;
};

/**
 * 统一 API 请求封装
 * @param {string} url - API 完整路径
 * @param {Object} options - 请求配置
 * @returns {Promise<Object>} API 响应数据
 */
export const fetchApi = async (url, options = {}) => {
  const { params, ...fetchOptions } = options;
  if (params) {
    const queryString = new URLSearchParams(params).toString();
    url = queryString ? `${url}?${queryString}` : url;
  }
  const response = await fetch(url, {
    ...fetchOptions,
    credentials: 'include',
    headers: {
      'Content-Type': 'application/json',
      ...fetchOptions.headers,
    },
  });
  return response.json();
};
```

### 3.3 公共组件规范

## 六、特殊规则

### 6.1 禁止使用useCallback

**任何情况下，都不要使用 useCallback**

所有函数使用普通箭头函数定义。

### 6.2 参数处理

**传入参数为对象时：**

不要在方法定义里将参数的属性展开显示，可以另起一行将要使用的对象属性展示。

```javascript
// ✅ 推荐
const loadData = async (params = {}) => {
  const { curPage, keyword, pageSize } = params;
  // 使用 curPage, keyword, pageSize
};

// ❌ 不推荐
const loadData = async ({ curPage, keyword, pageSize } = {}) => {
  // 直接使用参数
};
```

**方法需要传入3个或3个以上参数时：**

必须使用一个对象作为传入的参数。

```javascript
// ✅ 推荐
const handleAction = ({ record, action, callback }) => {
  // 处理逻辑
};

// ❌ 不推荐
const handleAction = (record, action, callback) => {
  // 处理逻辑
};
```

### 6.3 注释要求

**必须添加注释：**

- 重要函数必须有 JSDoc 注释
- 复杂逻辑必须有注释说明
- 代码块分组使用注释标识

**可以省略注释：**

- import 区域（除非有特殊说明需求）
- 简单的一目了然的代码

### 6.4 代码重复度控制

**index.js 代码生成时：**

1. 查找当前所有项目中的其他页面或组件文件
2. 要求不能和其他页面重复代码行超过10行
3. 如有重复，使用以下方法减少重复度：
   - 修改参数名称
   - 修改对象字段位置
   - 修改文件名称

**constant.js 代码生成时：**

1. 查看项目中其他文件是否有类似逻辑
2. 如有重复，将重复的地方列举出来
3. 询问是否要将逻辑抽取为通用方法放在项目的公共 constant 或 common 中

## 九、错误处理规范

### 9.1 API错误处理

**thunk.js 中必须使用 try-catch 包裹：**

```javascript
// ✅ thunk中必须使用try-catch包裹fetchApi
export const getXxxList = async (params = {}) => {
  try {
    const result = await fetchApi(API_CONFIG.XXX_LIST, { method: 'GET', params });
    return result || {};
  } catch (err) {
    return {};
  }
};
```

**index.js/组件中调用thunk方法时：**

```javascript
// 场景1：直接使用result，不需要额外try-catch
const result = await getXxxList(params);
if (result && result.code === '200') {
  message.success('操作成功');
} else {
  message.error(result?.messageZh || result?.message || '操作失败');
}

// 场景2：需要对result做数据处理时添加try-catch
const result = await getXxxDetail(id);
if (result && result.code === '200') {
  try {
    const configObj = JSON.parse(result.data.config);
    setConfig(configObj);
  } catch (parseError) {
    message.error('配置数据格式错误');
  }
}
```

### 9.2 统一提示信息

```javascript
// ✅ 使用 Ant Design 的 message 组件
import { message } from 'antd';

// 成功提示
message.success('操作成功');

// 失败提示：优先使用 messageZh，其次 message，最后使用固定文案
message.error(result?.messageZh || result?.message || '操作失败');

// 警告提示
message.warning('操作警告');

// 加载提示
message.loading({ content: '处理中...', key: 'action' });
```

### 检查清单

代码编写完成后，请检查：

#### 1. 文件结构检查

- [ ] 文件夹结构是否符合规范
- [ ] 每个页面是否单独一个文件夹
- [ ] 是否包含必要的文件：index.js、route.js、thunk.js、constant.js、样式文件

#### 2. 代码重复度检查

**所有文件（index.js、constant.js、thunk.js、组件文件）都需要检查：**

- [ ] 文件内部代码重复度是否超过10行
- [ ] 与其他页面文件代码重复度是否超过10行（例如：index.js 与其他页面的 constant.js 有10行以上重复）
- [ ] 如有重复，是否已通过以下方式减少重复度：
  - 修改参数名称
  - 修改对象字段位置
  - 修改文件名称
  - 抽取公共方法到 utils/common.js

#### 3. thunk.js 规范检查

- [ ] 所有方法是否都使用 try-catch 包裹 fetchApi
- [ ] 正常返回是否使用 `return result || {}`
- [ ] 异常返回是否使用 `return {}`

#### 4. constant.js 规范检查

- [ ] 是否有重复逻辑需要抽取
- [ ] 公共配置是否直接导入使用，而非重复导入导出
- [ ] 表格列配置是否使用统一的渲染函数

- [ ] 组件是否抽取到合适的目录
- [ ] 组件参数是否使用对象形式接收
- [ ] 组件注释是否完整

#### 6. 样式规范检查

- [ ] 是否使用 CSS Modules（.module.less）
- [ ] 是否避免行内样式
- [ ] 类名是否按照页面结构嵌套命名
- [ ] 样式类名是否与页面结构一致

#### 7. 其他检查

- [ ] 错误提示文案是否使用 `result?.messageZh || result?.message || 固定文案`
