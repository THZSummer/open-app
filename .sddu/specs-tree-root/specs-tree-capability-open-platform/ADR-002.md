# ADR-002: Mock 策略实现第三方依赖隔离

## 状态

ACCEPTED

## 背景

能力开放平台依赖多个现有系统：

| 依赖系统 | 用途 |
|----------|------|
| 应用管理系统 | 应用身份（AppID）、应用生命周期管理 |
| 凭证管理系统 | 应用凭证（AKSK、应用类凭证 A/B、开放应用凭证等） |
| 成员管理系统 | 用户身份、组织架构（含分组责任人） |
| XX通讯平台内部中台网关 | 转发三方平台 API 请求至消费网关进行鉴权 |
| 内部消息网关 | 转发业务模块产生的事件至消费网关 |
| 内部消息平台 | 事件分发、审批通知 |
| 操作日志系统 | 审计日志记录 |

在开发阶段，这些系统可能：
- 尚未提供稳定的测试环境
- 接口尚未完全确定
- 联调资源受限

## 决策

我们决定采用 **接口策略控制（Interceptor/Proxy 层）** 实现 Mock 与真实接口的 **一键切换**。

### 决策内容

#### 1. 抽象接口层

为每个外部依赖定义抽象接口：

```typescript
// 示例：应用管理接口
export interface IAppManagementService {
  getAppById(appId: string): Promise<App>;
  getAppList(params: AppListParams): Promise<App[]>;
  validateAppCredentials(appKey: string, appSecret: string): Promise<boolean>;
}
```

#### 2. 双实现

为每个接口提供两种实现：

```java
// Mock 实现
@Service
@Profile("mock")
public class MockAppManagementService implements IAppManagementService {
    @Override
    public App getAppById(String appId) {
        return mockApps.stream()
            .filter(app -> app.getId().equals(appId))
            .findFirst()
            .orElse(null);
    }
    // ...
}

// 真实实现
@Service
@Profile("!mock")
public class RealAppManagementService implements IAppManagementService {
    @Autowired
    private RestTemplate restTemplate;
    
    @Value("${external.app-management.base-url}")
    private String baseUrl;
    
    @Override
    public App getAppById(String appId) {
        return restTemplate.getForObject(baseUrl + "/apps/" + appId, App.class);
    }
    // ...
}
```

#### 3. 配置驱动切换

通过 Spring Profile 控制使用哪种实现：

```yaml
# application.yml
external:
  app-management:
    base-url: http://app-management.internal

# application-mock.yml (开发环境)
spring:
  profiles:
    active: mock
```

```java
// 配置类
@Configuration
public class ExternalServiceConfig {
    @Bean
    @Profile("mock")
    public IAppManagementService mockAppManagementService() {
        return new MockAppManagementService();
    }
    
    @Bean
    @Profile("!mock")
    public IAppManagementService realAppManagementService() {
        return new RealAppManagementService();
    }
}
```

#### 4. 环境配置

```yaml
# application-dev.yml (开发环境 - 全 Mock)
spring:
  profiles:
    active: mock,dev

# application-staging.yml (预发布环境 - 部分 Mock)
external:
  app-management:
    base-url: http://app-management.staging.internal
  member-management:
    base-url: http://member.staging.internal

# application-prod.yml (生产环境 - 无 Mock)
spring:
  profiles:
    active: prod
external:
  app-management:
    base-url: http://app-management.production.internal
  member-management:
    base-url: http://member.production.internal
```

### Mock 数据设计

Mock 数据应覆盖以下场景：

| 场景 | Mock 数据 |
|------|-----------|
| 正常数据 | 符合业务规则的正常数据 |
| 边界数据 | 空值、最大值、最小值 |
| 异常数据 | 错误码、超时、异常响应 |
| 并发场景 | 模拟并发请求 |

## 理由

### 为什么选择 Mock 策略

| 维度 | 优势 |
|------|------|
| **独立开发** | 无需等待外部系统就绪 |
| **快速验证** | 可快速验证业务逻辑 |
| **环境隔离** | 开发环境无需依赖外部系统 |
| **灵活切换** | 联调阶段一键切换真实接口 |
| **测试可控** | 可模拟各种异常场景 |

### 为什么不直接对接真实接口

| 维度 | 劣势 |
|------|------|
| **依赖阻塞** | 外部系统不稳定会阻塞开发 |
| **环境受限** | 测试环境资源有限 |
| **数据污染** | 开发测试可能污染真实数据 |
| **调试困难** | 外部系统问题难以定位 |

## 后果

### 正面影响

1. **开发效率提升**：无需等待外部系统就绪即可开始开发
2. **测试覆盖完整**：可模拟各种边界和异常场景
3. **降低联调成本**：Mock 通过后，联调仅验证接口契约
4. **环境独立**：每个开发者可在本地完整运行系统

### 负面影响

1. **Mock 与真实接口不一致风险**
   - **缓解措施**：
     - 定义接口契约（OpenAPI）
     - Mock 服务严格按契约实现
     - 契约测试验证一致性
     - 定期与真实接口对齐

2. **Mock 数据维护成本**
   - **缓解措施**：
     - 使用 Java Faker 或 Instancio 生成测试数据
     - 数据模板化管理
     - 定期更新 Mock 数据

### 契约测试策略

```java
@SpringBootTest
@ActiveProfiles("mock")
class AppManagementContractTest {
    
    @Autowired
    private IAppManagementService mockService;
    
    @Autowired
    private IAppManagementService realService;
    
    @Test
    void shouldMatchContractForGetAppById() {
        // Given
        String appId = "test-app-id";
        
        // When
        App mockResult = mockService.getAppById(appId);
        App realResult = realService.getAppById(appId);
        
        // Then - 验证返回结构一致
        assertThat(mockResult).isNotNull();
        assertThat(realResult).isNotNull();
        assertThat(mockResult.getId()).isInstanceOf(String.class);
        assertThat(realResult.getId()).isInstanceOf(String.class);
        // ...
    }
}
```
