## 2. 上下文
> 回顾问题背景和目标用户

本子 Feature 聚焦**面向能力消费方的开放面**，分为两大块：

### 2.1 后端层 — open-server ability 模块增强

现有能力：
- 能力列表查询 — 全量能力列表 + 已订阅标记
- 能力订阅 — 按能力类型编码订阅
- 已订阅列表查询 — 已订阅能力详情
- 订阅后自动桥接扩展点（空实现）

存在问题：
- 订阅校验通过硬编码类型枚举，自定义类型（平台面创建）无法通过校验
- 无取消订阅能力（订阅后无法退订）
- 自动桥接扩展点未实现，能力订阅后未与 API/事件权限系统打通
- 前端能力类型映射/场景分组为硬编码常量，自定义类型不展示

### 2.2 前端层 — wecodesite Capabilities 页增强

现有能力：
- 能力卡片网格展示（已订阅/未订阅切换）
- 添加能力 → 跳转配置页（目前展示占位文本 "配置页面由能力方提供"）
- 配置页无实际功能

存在问题：
- 能力类型映射为静态常量，仅含 7 种预设类型
- 场景分组为静态常量，自定义类型不在任何场景分组中
- 配置页无法加载嵌入能力方的实际 UI

### 2.3 目标用户

| 角色 | 说明 |
|------|------|
| **三方应用开发方** | 在 wecodesite 浏览能力目录、订阅/取消订阅能力、在配置页操作嵌入能力方提供的 UI |
| **嵌入能力方** | 提供微前端子应用入口，嵌入到 wecodesite 的配置页中 |
| **平台管理员** | 验证能力目录是否正确展示（不直接操作本面） |

## 3. 目标与非目标
> 明确需求范围，防止范围蔓延

### 3.1 目标 (Goals)
> 明确本次要达成的业务目标

| # | 目标描述 | 归属层 |
|---|---------|--------|
| G-001 | 三方应用可浏览完整 ability 目录（含预设 + 自定义类型） | 后端+前端 |
| G-002 | 三方应用可订阅/取消订阅 ability | 后端 |
| G-003 | 已订阅 ability 支持跳转到嵌入能力方提供的配置页面（微前端子应用） | 前端 |
| G-004 | 订阅 ability 后自动桥接 API/事件权限关联（自动桥接扩展点实现） | 后端 |
| G-005 | 前端能力目录动态化：不依赖硬编码常量，由后端数据驱动 | 前端 |

### 3.2 非目标 (Non-Goals)
> 明确本次不涉及的范围，防止需求蔓延

| # | 明确不做 | 原因 |
|---|---------|------|
| NG-001 | 不做能力使用统计与分析 | 属于进阶治理能力，后置 |
| NG-002 | 不修改能力开放平台的 API/事件/回调权限模型 | 通过自动桥接扩展点预留钩子，具体映射规则后置 |
| NG-003 | 不做微前端子应用沙箱/安全隔离方案 | 前端团队单独调研设计实施 |
| NG-004 | 不做依赖配置中心的动态场景分组 | 场景分组当前仍使用静态常量，未来可考虑动态化 |

## 4. 用户故事
> 以用户视角描述功能需求

| # | 作为… | 我想要… | 以便… |
|---|-------|---------|-------|
| US-001 | 三方应用开发方 | 在 wecodesite 的能力页面看到完整的能力目录（包括自定义类型） | 了解有哪些能力可用 |
| US-002 | 三方应用开发方 | 为我的应用订阅一个能力 | 启用该能力 |
| US-003 | 三方应用开发方 | 为我的应用取消订阅一个已订阅的能力 | 不再使用该能力 |
| US-004 | 三方应用开发方 | 在已订阅能力的配置页看到嵌入能力方提供的实际 UI | 完成该能力的配置操作（如设置群置顶内容） |
| US-005 | 嵌入能力方 | 通过提供微前端子应用入口，将配置 UI 嵌入 wecodesite | 让应用方可以配置我的能力 |

## 5. 功能需求 (FR)
> 每个需求必须有唯一标识符且可测试

### 5.1 开放面 — 后端（open-server ability 模块）

| ID | 需求描述 | 验收标准 | 优先级 |
|----|---------|---------|--------|
| FR-001 | **能力目录增强**：能力列表返回所有启用的 ability 类型（含预设 7 类和平台面自定义类型），并扩展返回字段 | • 返回所有启用的能力类型<br/>• 是否展示由后端基于配置控制（支持隐藏指定类型），前端不再硬编码过滤<br/>• 新增返回前端入口URL（有则返回，无则返回空）<br/>• 返回完整能力信息（已有字段保持向后兼容）<br/>• 预设类型可回退到本地常量做本地化展示；自定义类型直接使用后端返回信息 | P0 |
| FR-002 | **订阅校验增强**：订阅时校验能力类型是否存在，允许自定义类型 | • 移除硬编码类型枚举校验<br/>• 改为查询数据库校验能力类型存在且为启用状态<br/>• 自定义类型和预设类型走同一逻辑 | P0 |
| FR-003 | **取消订阅**：新增取消订阅指定能力 | • 按应用 + 能力类型取消订阅（软删除）<br/>• 取消后该能力在列表中标记为未订阅<br/>• 不可取消不存在的订阅记录 | P1 |
| FR-004 | **自动桥接扩展点实现**：订阅能力后触发自动桥接扩展点 | • 订阅能力后触发扩展点<br/>• 当前阶段输出日志记录即可<br/>• 预留钩子，后续可扩展为自动订阅对应的 API/事件权限 | P1 |
| FR-005 | **已订阅列表增强**：已订阅列表查询扩展返回字段 | • 新增返回前端入口URL<br/>• 确保能力名称等完整返回 | P0 |
| FR-006 | **能力隐藏控制**：支持设置某个能力类型在目录中隐藏 | • 隐藏的能力类型不出现在能力列表返回中<br/>• 已订阅的不受影响（已订阅列表正常返回） | P2 |

### 5.2 开放面 — 前端（wecodesite Capabilities 页）

| ID | 需求描述 | 验收标准 | 优先级 |
|----|---------|---------|--------|
| FR-101 | **动态能力目录**：能力卡片列表基于后端数据渲染，不依赖前端硬编码常量 | • 预设类型：保留现有展示逻辑（图标/名称可回退到常量）<br/>• 自定义类型：使用后端返回的信息渲染<br/>• 卡片按钮逻辑不变（未订阅→添加，已订阅→配置） | P0 |
| FR-102 | **已订阅能力配置页 — 嵌入子应用**：点击"配置"进入的能力配置页加载嵌入能力方提供的微前端子应用 | • 从前端入口URL获取子应用入口<br/>• 动态加载子应用到配置页容器<br/>• 向子应用传递上下文信息：应用ID、能力类型编码、能力中文名<br/>• 子应用可独立运行（拥有独立路由/状态）<br/>• 无前端入口URL的能力仍展示占位文本 | P0 |
| FR-103 | **能力场景分组动态化**（可选）：支持由后端控制能力的场景分组 | • 保留现有静态场景分组作为兜底<br/>• 可扩展为后端返回分组元信息（当前阶段保持静态） | P2 |

### 5.3 业务接口

| 业务操作 | 变更类型 | 变更说明 |
|---------|---------|---------|
| 能力列表查询 | **修改** | 新增前端入口URL字段；移除前端硬编码过滤；不依赖前端常量兜底 |
| 能力订阅 | **修改** | 校验逻辑改为查数据库 |
| 取消订阅 | **新增** | 按应用 + 能力类型取消订阅 |
| 已订阅列表查询 | **修改** | 新增前端入口URL等字段 |

### 5.4 业务字段

**能力类型新增字段**（与平台面共享）：

| 业务字段 | 说明 | 归属 |
|---------|------|------|
| 前端入口URL | 微前端子应用入口地址 | 平台面写入，开放面读取 |
| 是否在开放面展示 | 控制是否在目录中展示（默认展示） | 平台面管理，开放面过滤 |

### 5.5 前端嵌入协议

**微前端子应用约定**（嵌入能力方需遵守）：

**交互流程**：
1. 用户点击"配置"按钮
2. 主应用读取前端入口URL
3. 动态注册并挂载子应用到配置页容器，传递上下文信息
4. 子应用渲染配置 UI
5. 用户操作完成后，主应用卸载子应用

**上下文信息**：

| 信息 | 说明 | 示例 |
|------|------|------|
| 应用ID | 当前应用 ID | `wx_app_001` |
| 能力类型编码 | 能力类型编码 | `100` |
| 能力中文名 | 能力中文名 | `群置顶服务` |

**子应用要求**：
- 支持微前端生命周期（加载/挂载/卸载）
- 独立运行时也能正常开发调试
- UI 风格尽量与主应用保持一致

## 6. 非功能需求 (NFR)
> 性能、安全、可用性等跨切面需求

| ID | 类别 | 需求描述 | 验收标准 |
|----|------|---------|---------|
| NFR-001 | 性能 | 能力列表查询响应时间 | P99 < 200ms |
| NFR-002 | 性能 | 能力订阅/取消订阅响应时间 | P99 < 500ms |
| NFR-003 | 可用性 | 嵌入的微前端子应用加载超时处理 | 子应用加载超过 10s 展示加载失败提示，不影响其他功能 |
| NFR-004 | 兼容性 | 响应向后兼容 | 新增字段为可选，不破坏现有前端代码 |
| NFR-005 | 可用性 | 取消订阅前二次确认 | 弹窗确认："取消订阅后，该能力的配置数据可能会丢失，确定取消吗？" |

## 7. 边界情况 (EC)
> 异常场景和边界条件的处理方式

| ID | 场景 | 处理方式 |
|----|------|---------|
| EC-001 | 取消订阅后误操作再次订阅同一能力 | 走正常订阅流程，重新创建订阅记录（幂等处理） |
| EC-002 | 前端入口URL对应的微前端子应用不可用（服务器宕机/URL 错误） | 配置页展示加载失败提示，不阻塞其他功能 |
| EC-003 | 子应用加载时上下文信息缺失 | 主应用确保必填信息完整性，缺失时展示"配置加载失败，请重试" |
| EC-004 | 同一个能力同时被多个应用订阅/取消订阅 | 按应用隔离，互不影响 |
| EC-005 | 用户同时打开多个能力的配置页 | 每次只有一个子应用处于活跃状态（切换时自动卸载上一个） |

## 8. 开放问题
> 待决策事项和需要进一步调研的内容

| # | 问题 | 影响范围 | 建议方案 | 状态 |
|---|------|---------|---------|:----:|
| 1 | 微前端子应用如何动态注册？当前为静态配置，配置页需要运行时动态加载 | 前端架构 | 在配置页组件内运行时动态注册，或提供标准注册入口 | ⏳ 待前端调研 |
| 2 | 自动桥接扩展点具体的权限映射规则是什么？一个能力类型对应哪些权限？ | 后端架构 | 当前阶段仅打日志+预留钩子，具体映射规则需在权限模型基础上定义 | ⏳ 待确认 |
| 3 | 已订阅的能力类型被平台面删除后是否通知应用方？ | 治理 | MVP 阶段不做通知（平台面删除前已有订阅检查），后续可增量补充 | ⏳ 待决策 |
