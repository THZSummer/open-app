## 2. 上下文
> 回顾问题背景和目标用户

本子 Feature 聚焦**面向能力消费方的开放面**，核心变更范围：
- **后端**：能力列表查询增强（支持自定义类型 + 隐藏控制），订阅校验增强（枚举→查库）。订阅写入逻辑基本不变。
- **前端**：配置页从占位文本改为实际加载嵌入能力方提供的微前端子应用。

### 2.1 后端层 — open-server ability 模块增强

**现有逻辑**：

| 操作 | 现状 |
|------|------|
| 能力列表查询 | 从数据库查询所有 ability，排除 type=6，标记已订阅状态，返回名称/描述/图标/示意图等 |
| 能力订阅 | 校验 abilityType 是否在硬编码枚举中 → 检查重复 → 插入订阅关联记录 → 触发自动桥接扩展点（空实现） |
| 已订阅列表查询 | 查询应用的订阅记录，排除 type=6，关联 ability 表返回名称/图标/排序等 |

**存在问题**：
- 订阅校验通过硬编码类型枚举，平台面创建的自定义类型无法通过校验
- type=6 通过代码硬编码排除，新增的隐藏类型需要改代码
- 自动桥接扩展点未实现，能力订阅后未与 API/事件权限系统打通
- 列表/已订阅列表缺少前端入口URL字段

### 2.2 前端层 — wecodesite Capabilities 页增强

**现有逻辑**：

| 操作 | 现状 |
|------|------|
| 能力卡片展示 | 场景分组 tabs + 能力卡片网格，卡片渲染从后端数据取名称/描述/图标，标识已订阅/未订阅状态 |
| 订阅操作 | 点击"添加"→ 调用订阅 API → 成功后自动跳转到配置页 |
| 配置页入口 | URL `?appId=X&sub=<abilityType>` 已支持切换视图，但当前展示占位文本"配置页面由能力方提供" |
| 常量 | 前端有 ABILITY_TYPE_MAP（中文名/图标映射）和 ABILITY_SCENE_MAP（场景分组）两个静态常量 |

**存在问题**：
- 前端常量仅含 7 种预设类型，自定义类型无对应中文名/图标/场景分组
- 配置页为占位文本，未实际加载嵌入能力方的 UI

### 2.3 目标用户

| 角色 | 说明 |
|------|------|
| **三方应用开发方** | 在 wecodesite 浏览能力目录、订阅能力、在配置页操作嵌入能力方提供的 UI |
| **嵌入能力方** | 提供微前端子应用入口，嵌入到 wecodesite 的配置页中 |
| **平台管理员** | 验证能力目录是否正确展示（不直接操作本面） |

## 3. 目标与非目标
> 明确需求范围，防止范围蔓延

### 3.1 目标 (Goals)
> 明确本次要达成的业务目标

| # | 目标描述 | 归属层 |
|---|---------|--------|
| G-001 | 三方应用可浏览完整 ability 目录（含预设 + 自定义类型，受隐藏配置控制） | 后端+前端 |
| G-002 | 三方应用可订阅 ability（支持自定义类型校验） | 后端 |
| G-003 | 已订阅 ability 支持按 URL 地址打开嵌入能力方提供的配置页面（微前端子应用） | 前端 |
| G-004 | 订阅 ability 后自动桥接 API/事件权限关联（自动桥接扩展点实现） | 后端 |
| G-005 | 前端能力目录动态化：自定义类型使用后端返回数据渲染，不依赖前端硬编码常量 | 前端 |

### 3.2 非目标 (Non-Goals)
> 明确本次不涉及的范围，防止需求蔓延

| # | 明确不做 | 原因 |
|---|---------|------|
| NG-001 | 不做能力使用统计与分析 | 属于进阶治理能力，后置 |
| NG-002 | 不做取消订阅能力 | 现有流程无取消订阅场景，后续有需求可增量补充 |
| NG-003 | 不修改能力开放平台的 API/事件/回调权限模型 | 通过自动桥接扩展点预留钩子，具体映射规则后置 |
| NG-004 | 不做微前端子应用沙箱/安全隔离方案 | 前端团队单独调研设计实施 |
| NG-005 | 不做依赖配置中心的动态场景分组 | 场景分组当前仍使用静态常量，未来可考虑动态化 |

## 5. 功能需求 (FR)
> 每个需求必须有唯一标识符且可测试

### 5.1 开放面 — 后端（open-server ability 模块）

| ID | 需求描述 | 验收标准 | 优先级 |
|----|---------|---------|--------|
| FR-001 | **能力列表增强**：增强能力列表查询，支持返回自定义类型 + 前端入口URL + 隐藏控制 | • 返回数据库中的所有启用能力类型（不再通过代码硬编码排除特定 type）<br/>• 根据平台面的 hidden 字段决定是否在列表中展示（hidden=1 的不出现）<br/>• 新增返回前端入口URL（有则返回，无则返回空）<br/>• 返回完整能力信息（名称、描述、图标、示意图、排序等，已有字段保持向后兼容）<br/>• 已订阅标记逻辑不变 | P0 |
| FR-002 | **订阅校验增强**：订阅时校验能力类型是否存在（支持自定义类型） | • 移除硬编码类型枚举校验<br/>• 改为查询数据库校验能力类型存在且为启用状态<br/>• 重复订阅检查、插入关联记录等其余逻辑不变 | P0 |
| FR-003 | **自动桥接扩展点实现**：订阅能力后触发自动桥接扩展点 | • 订阅能力后触发扩展点<br/>• 当前阶段输出日志记录即可<br/>• 预留钩子，后续可扩展为自动订阅对应的 API/事件权限 | P1 |
| FR-004 | **已订阅列表增强**：已订阅列表查询扩展返回字段 | • 新增返回前端入口URL<br/>• 确保能力名称等完整返回<br/>• 其余逻辑不变 | P0 |
| FR-005 | **能力隐藏控制**：支持平台面设置某个能力类型在目录中隐藏 | • 读取平台面的 hidden 字段<br/>• 隐藏的能力类型不出现在能力列表返回中<br/>• 已订阅的不受影响（已订阅列表正常返回）<br/>• 替换现有的代码硬编码排除逻辑 | P2 |

### 5.2 开放面 — 前端（wecodesite Capabilities 页）

| ID | 需求描述 | 验收标准 | 优先级 |
|----|---------|---------|--------|
| FR-101 | **动态能力目录**：能力卡片列表基于后端数据渲染，自定义类型不依赖前端常量 | • 预设类型：保留现有展示逻辑（图标/中文名可回退到常量作为兜底）<br/>• 自定义类型：直接使用后端返回的中文名/描述/图标渲染<br/>• 场景分组：自定义类型归入默认场景（或由 ABILITY_SCENE_MAP 扩展配置）<br/>• 卡片按钮逻辑不变（未订阅→添加，已订阅→配置） | P0 |
| FR-102 | **配置页嵌入子应用**：URL `?appId=X&sub=<abilityType>` 进入的配置页加载嵌入能力方提供的微前端子应用 | • 从后端返回的前端入口URL获取子应用入口地址<br/>• 在配置页容器中动态加载并挂载子应用<br/>• 向子应用传递上下文信息：应用ID、能力类型编码、能力中文名<br/>• 子应用可独立运行（拥有独立路由/状态）<br/>• 无前端入口URL的能力仍展示占位文本 | P0 |
| FR-103 | **能力场景分组动态化**（可选）：支持由后端控制能力的场景分组 | • 保留现有静态场景分组作为兜底<br/>• 可扩展为后端返回分组元信息（当前阶段保持静态） | P2 |

### 5.3 业务接口

| 业务操作 | 变更类型 | 变更说明 |
|---------|---------|---------|
| 能力列表查询 | **修改** | 新增前端入口URL字段；根据 hidden 字段控制展示；不再代码硬编码排除特定 type |
| 能力订阅 | **修改** | 校验逻辑改为查数据库（其余逻辑不变） |
| 已订阅列表查询 | **修改** | 新增前端入口URL等字段 |

### 5.4 业务字段

**能力类型新增字段**（与平台面共享）：

| 业务字段 | 说明 | 归属 |
|---------|------|------|
| 前端入口URL | 微前端子应用入口地址 | 平台面写入，开放面读取 |
| 是否在开放面展示 | 控制是否在目录中展示（默认展示） | 平台面管理，开放面过滤 |

### 5.5 前端嵌入协议

**微前端子应用约定**（嵌入能力方需遵守）：

**交互流程**：
1. 用户点击已订阅能力的"配置"按钮，或直接访问配置页 URL
2. 主应用读取该能力的前端入口URL
3. 动态加载子应用到配置页容器，传递上下文信息
4. 子应用渲染配置 UI
5. 用户操作完成后（或切换能力），主应用卸载子应用

**上下文信息**：

| 信息 | 说明 | 示例 |
|------|------|------|
| 应用ID | 当前应用 ID | `wx_app_001` |
| 能力类型编码 | 能力类型编码 | `100` |
| 能力中文名 | 能力中文名 | `群置顶服务` |

**子应用要求**：
- 支持微前端生命周期（加载/挂载/卸载）
- 独立运行时也能正常开发调试
- UI 风格尽量与主应用保持一致

## 6. 非功能需求 (NFR)
> 性能、安全、可用性等跨切面需求

| ID | 类别 | 需求描述 | 验收标准 |
|----|------|---------|---------|
| NFR-001 | 性能 | 能力列表查询响应时间 | P99 < 200ms |
| NFR-002 | 性能 | 能力订阅响应时间 | P99 < 500ms |
| NFR-003 | 可用性 | 嵌入的微前端子应用加载超时处理 | 子应用加载超过 10s 展示加载失败提示，不影响其他功能 |
| NFR-004 | 兼容性 | 响应向后兼容 | 新增字段为可选，不破坏现有前端代码 |
| NFR-005 | 可用性 | 订阅前确认 | 非必要不弹窗；仅当覆盖已有配置数据时需二次确认 |

## 7. 边界情况 (EC)
> 异常场景和边界条件的处理方式

| ID | 场景 | 处理方式 |
|----|------|---------|
| EC-001 | 能力被平台面设为隐藏后，已订阅用户进入配置页 | 配置页正常展示（已订阅不受隐藏影响），但新用户无法在列表中看到 |
| EC-002 | 前端入口URL对应的微前端子应用不可用（服务器宕机/URL 错误） | 配置页展示加载失败提示，不阻塞其他功能 |
| EC-003 | 子应用加载时上下文信息缺失 | 主应用确保必填信息完整性，缺失时展示"配置加载失败，请重试" |
| EC-004 | 同一个能力同时被多个应用订阅 | 按应用隔离，互不影响 |
| EC-005 | 用户同时打开多个能力的配置页 | 每次只有一个子应用处于活跃状态（切换时自动卸载上一个） |

## 8. 开放问题
> 待决策事项和需要进一步调研的内容

| # | 问题 | 影响范围 | 建议方案 | 状态 |
|---|------|---------|---------|:----:|
| 1 | 微前端子应用如何动态注册？当前为静态配置，配置页需要运行时动态加载 | 前端架构 | 在配置页组件内运行时动态注册，或提供标准注册入口 | ⏳ 待前端调研 |
| 2 | 自动桥接扩展点具体的权限映射规则是什么？一个能力类型对应哪些权限？ | 后端架构 | 当前阶段仅打日志+预留钩子，具体映射规则需在权限模型基础上定义 | ⏳ 待确认 |
| 3 | 自定义类型的场景分组如何确定？前端 ABILITY_SCENE_MAP 仅含预设类型 | 前端 | 自定义类型默认归入"其他"场景或扩展 ABILITY_SCENE_MAP | ⏳ 待决策 |
