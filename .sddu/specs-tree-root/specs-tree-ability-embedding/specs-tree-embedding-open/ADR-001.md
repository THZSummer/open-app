# ADR-001: 增量修改现有 ability 模块而非新建模块

## 状态
ACCEPTED

## 背景
- 开放面对后端的变更范围：列表增强（hidden 过滤 + entryUrl/routePath/aliasName/requireRelease）、订阅校验（枚举→DB）、已订阅列表增强（entryUrl/routePath/aliasName/requireRelease）
- 可被现有 `AbilityServiceImpl` 的三个方法容纳，无实质新增接口

## 决策
直接修改 `AbilityServiceImpl` 现有方法，VO 类新增 optional 字段 `entryUrl`、`routePath`、`aliasName`、`requireRelease`。不创建新 Controller 或新 Service。

## 后果
### 正面
- 最小代码变更
- 接口向后兼容，新增字段为 optional
- 复用现有测试和部署链路

### 负面
- 旧硬编码逻辑（type=6 排除）需谨慎替换为 `hidden` 字段，确保语义等价
