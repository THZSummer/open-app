# ADR-002: 配置页动态子应用使用运行时 loadMicroApp

## 状态
ACCEPTED

## 背景
- 现有 `microApps.js` 使用静态注册（应用启动时注册所有子应用）
- 能力配置页的子应用 `frontendEntryUrl` 由平台面运行时录入，运行时才能确定
- 不需要在导航栏或全局路由中注册该子应用

## 决策
在 Capabilities 页配置视图组件内，使用 QianKun 的 `loadMicroApp` API 运行时动态加载子应用。不修改 `microApps.js`。

## 后果
### 正面
- 不污染全局注册列表，切换能力时自动 unload
- 动态加载，无 `frontendEntryUrl` 的能力不走加载流程

### 负面
- 依赖 wecodesite 的 QianKun 版本是否支持 `loadMicroApp` API
- 如不支持，需使用 `registerMicroApps` + `start` 的另一种模式
