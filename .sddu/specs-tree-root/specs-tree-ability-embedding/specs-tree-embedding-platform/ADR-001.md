# ADR-001: Admin 能力类型 CRUD 放在 open-server 扩展

## 状态
ACCEPTED

## 背景
- `openplatform_ability_t` 表位于 open-server 的 schema 中
- open-server ability 模块已有 `AbilityMapper`、`AbilityPropertyMapper`、`AbilityService`、`fileV2Service` 等完整组件
- 规范要求"写入即对开放面可见"（NFR-003），开放面和平台面读写同一数据源最直接
- 规范写"服务端：market-server"，但 ability 数据表在 open-server

## 决策
在 open-server 的 `ability` 模块内新增 `AdminAbilityController` + `AdminAbilityService`（新类，不修改现有 AbilityController/AbilityService）。market-web 作为前端直接调用 open-server 的 admin 接口前缀 `/service/open/v2/ability/admin/`。

## 后果
### 正面
- 复用现有 Mapper/Entity/Service，不引入跨服务调用
- 写入即时对开放面可见（同一 DB）
- 不修改现有 AbilityController，不影响开放面已有接口

### 负面
- 与 spec 中"服务端：market-server"的表述不一致，需在 plan 中说明实际调整为 open-server
