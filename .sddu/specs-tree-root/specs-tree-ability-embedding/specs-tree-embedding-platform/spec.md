## 2. 上下文
> 回顾问题背景和目标用户

基于父 Feature 问题挖掘报告，本子 Feature 聚焦**平台管理面**：

**要解决的问题**：当前 ability 类型由 `AbilityTypeEnum` 硬编码预置（7 种），平台管理员无法通过管理后台新增/编辑/排序 ability 类型。业务模块要嵌入新能力，必须依赖平台方修改代码重新部署。

**目标用户**：
- **平台管理员**（开放平台管理员）— 在 market-web 后台维护 ability 目录

**业务背景**：嵌入能力方（IM、云盘等业务模块）需要将其特有连接能力注册到能力开放平台。平台面作为"录入端"，由平台管理员在管理后台创建 ability 类型，录入完整信息后发布到目录。能力方不直接操作平台面，通过线下/线上沟通告知平台管理员需求后由平台管理员录入。

**关联**：
- 下游依赖：嵌入能力开放面（`EMBED-OPEN-001`）消费本平台面发布的能力目录
- 已存在代码：open-server `ability` 模块的 `Ability`、`AbilityProperty` 实体

## 5. 功能需求 (FR)
> 每个需求必须有唯一标识符且可测试

### 5.1 能力管理

| ID | 需求描述 | 验收标准 | 优先级 |
|----|---------|---------|--------|
| FR-001 | **能力目录列表**：平台管理员在 market-web 查看所有 ability 类型列表 | • 列表展示字段：abilityType 编码、中文名、英文名、描述、图标、示意图缩略图、排序号、加载类型、前端入口URL、创建时间、更新人、更新时间<br/>• 分页展示，默认按排序号升序 | P0 |
| FR-002 | **创建能力**：平台管理员创建新的 ability 类型 | • 表单字段详见 §5.2 字段约束表（共 14 个字段）<br/>• abilityType 编码需校验唯一性（不与已有编码冲突）<br/>• 加载类型校验：loadType=2（微前端加载）时，entryUrl/routePath/aliasName 三要素必填<br/>• 图标/示意图以 batchId 形式提交（由 FR-005 通用文件上传接口返回）<br/>• 创建后状态默认为「启用」<br/>• 创建后同步写入 `openplatform_ability_t` 主表和 `openplatform_ability_p_t` 属性表 | P0 |
| FR-003 | **编辑能力**：平台管理员修改已有 ability 类型的信息 | • 支持修改 FR-002 中所有字段（含 loadType）<br/>• 修改时传入的字段同样受 §5.2 约束（名称 2-30 字符、描述 5-200 字符、图标格式/尺寸/大小、排序 ≥1 等）<br/>• abilityType 编码不可修改（作为业务标识）<br/>• 图标/示意图以 batchId 形式提交（由 FR-005 通用文件上传接口返回），新 batchId 覆盖旧引用<br/>• `hidden` 字段可切换（展示/隐藏状态）<br/>• 当 loadType 改为 2 时，entryUrl/routePath/aliasName 三要素必填 | P0 |
| FR-004 | **删除能力**：平台管理员删除 ability 类型 | • 删除前检查是否有应用订阅了该能力<br/>• 有关联订阅 > 禁止删除，提示"该能力已被 XX 个应用订阅，无法删除"<br/>• 无关联订阅 > 允许删除 | P1 |

### 5.2 文件上传

| ID | 需求描述 | 验收标准 | 优先级 |
|----|---------|---------|--------|
| FR-005 | **通用文件上传**：平台管理员在创建/编辑能力时统一通过此接口上传图标或示意图 | • 接口路径: `POST /service/open/v2/ability/admin/upload`<br/>• 请求参数: `file`（MultipartFile）、`bizType`（int，1=能力图标，2=能力示意图）<br/>• 图标校验（bizType=1）: 仅允许 PNG/SVG 格式，尺寸固定 40×40PX，文件大小 ≤200KB<br/>• 示意图校验（bizType=2）: 仅允许 PNG/JPG 格式，尺寸固定 520×288PX，文件大小 ≤500KB<br/>• 校验失败返回 400，提示具体规则（格式不符/尺寸不符/文件过大）<br/>• 校验通过后写入 `openplatform_common_file_t` 表（开发环境临时表），返回 `{batchId, showUrl}`<br/>• 接口权限：仅平台管理员可调用 | P0 |

### 5.3 字段约束

> 下表汇总所有能力表单字段的业务约束（含已有字段补充 + 新增字段）。
> 图标/示意图实际文件存储在 `openplatform_ability_p_t` 属性表，主表仅存 URL 引用。

| # | 字段 | DB列 | 类型 | 必填 | 约束 |
|---|------|------|------|:--:|------|
| 1 | 中文名称 | `ability_name_cn` | string | ✅ | 2-30 字符 |
| 2 | 英文名称 | `ability_name_en` | string | ✅ | 2-30 字符 |
| 3 | 中文描述 | `ability_desc_cn` | string | ✅ | 5-200 字符 |
| 4 | 英文描述 | `ability_desc_en` | string | ✅ | 5-200 字符 |
| 5 | 图标 | 属性表（`icon`） | batchId | ✅ | PNG / SVG，40×40PX，≤200KB；batchId 由 FR-005 上传接口返回 |
| 6 | 示意图 | 属性表（`diagram`） | batchId | ❌ | PNG / JPG，520×288PX，≤500KB；batchId 由 FR-005 上传接口返回，未上传时前端不展示 |
| 7 | 排序值 | `order_num` | int | ✅ | 正整数 ≥1，默认当前最大值 +1，支持手动输入和加减按钮 |
| 8 | 能力编码 | `ability_type` | int | ✅ | 创建时手动输入，1-7 预置保留，自定义 8-255，唯一校验，创建后不可修改 |
| 9 | 加载类型 | `load_type` | tinyint | ❌ | 1=路由加载（默认），2=微前端加载（QianKun loadMicroApp） |
| 10 | 进入地址 | `entry_url` | string | ❌ | HTTP/HTTPS 协议，≤1000 字符；load_type=2 时必填 |
| 11 | 路由路径 | `route_path` | string | ❌ | 子应用激活路由（QianKun activeRule）；load_type=2 时必填 |
| 12 | 别名 | `alias_name` | string | ❌ | 子应用唯一标识（QianKun name）；load_type=2 时必填 |
| 13 | 是否在开放面展示 | `hidden` | tinyint | ❌ | 0=展示，1=隐藏（默认隐藏） |
| 14 | 需版本发布 | `require_release` | tinyint | ❌ | 0=即时生效（默认），1=需版本发布 |

**abilityType 编码规则**：
- 1-7：保留给现有预置能力（群置顶、群通知、链接增强、点对点通知、we码、应用入群通知、助手广场卡片）
- 自定义类型统一在 tinyint 范围内分配，不区分类型大小
- 创建时校验编码唯一性（包括已创建的记录和预置编码）

## 6. 非功能需求 (NFR)
> 性能、安全、可用性等跨切面需求

| ID | 类别 | 需求描述 | 验收标准 |
|----|------|---------|---------|
| NFR-001 | 安全性 | 平台面操作仅限平台管理员 | 接口需校验当前用户角色，非管理员返回 403。备注：平台管理员角色校验沿用现有的角色权限校验逻辑即可，不需新建。 |
| NFR-002 | 可用性 | 图标/示意图上传校验：FR-005 通用文件上传接口在服务端校验文件格式/尺寸/大小 | • 图标: PNG/SVG，40×40PX，≤200KB<br/>• 示意图: PNG/JPG，520×288PX，≤500KB<br/>• 前端可选二次校验，以服务端校验为准 |
| NFR-003 | 一致性 | 平台面创建/编辑的 ability 类型即时同步到开放面 | 开放面查询接口直接读库，无需缓存刷新，写入即对开放面可见 |
| NFR-004 | 审计 | 关键操作记录审计日志 | 创建/编辑/删除操作需记录操作用户、时间、变更内容 |

## 7. 边界情况 (EC)
> 异常场景和边界条件的处理方式

| ID | 场景 | 处理方式 |
|----|------|---------|
| EC-001 | 创建时 abilityType 编码与已有记录或枚举值冲突 | 系统校验唯一性，冲突时提示"编码 XX 已被占用，请使用其他编码" |
| EC-002 | 删除时该类型仍有应用订阅 | 禁止删除，返回关联订阅数量及提示引导 |
| EC-003 | 前端入口 URL 格式不合法 | 创建/编辑时校验 URL 格式（http/https 协议），不合法则拒绝保存 |
| EC-004 | 正在被编辑时，有其他管理员同时操作 | 乐观锁处理（基于 `last_update_time` 或版本号），冲突时提示"数据已被修改，请刷新后重试" |
| EC-005 | 创建/编辑时未上传图标或图标格式/尺寸不符合要求 | 前端校验拦截（图标为必填项），不符合 PNG/SVG、40×40PX、≤200KB 时提示具体规则 |
| EC-006 | 排序号重复 | 允许重复输入，列表按 order_num + id 组合排序（同 order_num 按 id 升序） |
| EC-007 | 上传文件格式/尺寸/大小不符合要求 | FR-005 接口校验失败，返回 400；图标提示 "请上传 PNG/SVG 格式，40×40PX，不超过 200KB"；示意图提示 "请上传 PNG/JPG 格式，520×288PX，不超过 500KB" |
