# ADR-002: 应用标识解析采用识别器模式

## 状态
ACCEPTED

## 背景
- 嵌入能力方调用接口时传入的应用标识可能是平台 `appId` 或外部 `hisAppId`
- 系统需要自动判断类型并完成解析，对调用方透明

## 决策
创建 `AppIdentifierResolver`，解析策略：
1. 先尝试按平台 `appId` 查 `app_t` 表
2. 匹配失败则按 `hisAppId`（`eamap_app_code`）查 `app_p_t` 表
3. 都失败则返回"应用不存在"
最终统一为内部 `appId` 后向下传递。

## 后果
### 正面
- 调用方不感知标识类型差异，接口统一
- 识别逻辑集中，便于维护和扩展

### 负面
- 最坏情况下需两次 DB 查询（可通过前缀规则优化：数字+短横线格式判定为平台 appId）
