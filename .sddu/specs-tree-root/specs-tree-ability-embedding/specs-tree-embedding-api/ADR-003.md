# ADR-003: 内部凭证鉴权方案

## 状态
PROPOSED

## 背景
- API 面接口仅限内部服务调用，需要内部凭证鉴权
- 凭证由平台管理员预配置

## 决策
- MVP 阶段：内部凭证配置在 `application.yml` 中（静态配置）
- 凭证通过请求头传递（如 `X-Internal-Token`）
- 实现为 `HandlerInterceptor` 或 `Filter`，拦截 `/internal/` 路径
- 支持多服务方独立凭证列表（yml 中配置多个 token）
- 开发阶段可配置 bypass

## 后果
### 正面
- 实现简单，MVP 阶段快速可用

### 负面
- 静态配置，凭证变更需重启服务（后续可加管理 API）
