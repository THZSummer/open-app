# ADR-001: 新建独立 UserRoleController

## 状态
ACCEPTED

## 背景
- 用户角色查询是 API 面为嵌入能力方新增的核心接口
- 接口仅限内部服务调用，使用 `/internal/` 路径前缀
- 现有 `ScopeController` 负责用户授权管理，与角色查询职责不同
- 现有 `ApiGatewayController` 负责应用认证，也与角色查询不同

## 决策
新建 `UserRoleController`，包路径 `.../api/internal/controller/`。接口路径 `/internal/user/roles`。Service 层支持 Mock/Real 策略切换。

## 后果
### 正面
- 单一职责，职责清晰
- 内部接口与对外接口路径分离
- 可独立进行版本演进和测试

### 负面
- 新增一个 Controller，代码量增加
