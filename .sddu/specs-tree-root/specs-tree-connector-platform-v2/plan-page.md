## 2. 核心页面详设

### 2.1 连接器版本历史面板（VersionHistoryPanel.jsx）

**入口**: 连接器编辑页 → 「版本历史」Tab  
**对应 FR**: FR-005 ~ FR-011

```
┌─────────────────────────────────────────────────────────────┐
│ 版本历史                              [草稿 v3] [发布草稿]    │
├─────────────────────────────────────────────────────────────┤
│ ┌───────────────────────────────────────────────────────────┐│
│ │ 版本号  状态      发布时间        发布人      操作        ││
│ │ v3     📝草稿     —              —         [编辑] [发布] ││
│ │ v2     ✅已发布   06-09 10:00    zhangsan  [查看] [复制到草稿]│
│ │ v1     ❌已失效   06-08 09:00    lisi      [查看] [恢复] [删除]│
│ └───────────────────────────────────────────────────────────┘│
│                                    版本上限：3/1000           │
└─────────────────────────────────────────────────────────────┘
```

**交互流程**:
1. 版本列表展示所有版本（草稿 + 已发布 + 已失效）
2. 点击版本行 → 右侧展示该版本配置快照（只读）
3. 草稿行显示 [编辑] [发布] 按钮
4. 已发布行显示 [查看] [复制到草稿]（达上限时置灰 + Tooltip）
5. 已失效行显示 [查看] [恢复] [删除]
6. [复制到草稿] → 校验版本数上限 → 有草稿则覆盖确认弹窗 → 创建/覆盖草稿
7. [发布] → 校验草稿非空 → 调用发布接口 → 状态变更为已发布

#### 2.6.1 并行边切换

```

┌─────────────────────────┐
```

- React Flow 的 `edge.data.connectionMode = "serial" | "parallel"`

#### 2.6.2 连接器版本选择（VersionSelectPanel.jsx）

```
添加连接器节点 → 选择连接器 → 弹出版本选择面板:
┌─────────────────────────────────────────────┐
│ 选择连接器版本 — IM 发送消息                  │
├─────────────────────────────────────────────┤
│ ○ v3  ✅已发布  2026-06-09  zhangsan        │
│ ● v2  ✅已发布  2026-06-08  lisi   [当前选中]│

│                              [确认] [取消]   │
└─────────────────────────────────────────────┘
```

- 默认选中最新已发布版本
- 已失效版本灰显不可选
- 选择后节点显示版本标签（如 `v2`）

#### 2.6.3 数据处理节点（DataProcessorNode.jsx）

```
画布左侧节点面板 → 新增「数据处理」节点（可拖入画布）
选中节点 → 右侧属性面板:
┌─────────────────────────────────────────────┐
│ 数据处理节点                                 │
├─────────────────────────────────────────────┤
│ 输出字段:                         [+ 添加]  │
│ ┌─────────────────────────────────────────┐ │
│ │ 字段名: [userId            ]            │ │
│ │ 类型:   [string ▼]                      │ │
│ │ 值来源: [函数输出 ▼]                     │ │
│ │ ┌─ 函数配置 ─────────────────────────┐  │ │
│ │ │ 函数: [toString ▼]                 │  │ │
│ │ │ 入参: [${$.node.trigger.input.userId}] ││
│ │ │       [引用字段 ▼]  [选择字段...]    │  │ │
│ │ └────────────────────────────────────┘  │ │
│ └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

**值来源三种模式**:
- **静态值** → 直接输入文本框
- **引用字段** → 弹出字段选择器（显示上游所有节点的基本类型字段）
- **函数输出** → 选择函数 → 配置入参（入参同样支持三种来源，可递归嵌套）

**字段类型约束（FR-047）**:
- 选择 `object` 类型 → 必须展开定义子字段（标红提示）
- 选择 `array` 类型 → 必须声明 items 类型（标红提示）
- 引用路径终点为 `object`/`array` → 标红禁止保存
- 引用源类型与目标类型不一致 → 标红提示
