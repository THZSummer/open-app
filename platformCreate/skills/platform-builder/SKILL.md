# Platform Builder - 平台搭建器

## 技能概述

这个技能用于根据用户提供的平台规格文档，自动查找、分析并生成详细的可执行 task，然后严格按照 task 搭建完整的平台网站。

## 使用场景

当用户需要：
- 根据已有的平台规格文档搭建网站
- 自动生成详细的执行 task
- 严格按照 task 执行并验证

## 执行流程

### Phase 1: 查找规格文档

**自动查找项目根目录下的规格文档：**

1. **搜索常见命名模式的规格文档**
   - `spec*.md`
   - `*spec*.md`
   - `*规格*.md`
   - `specification*.md`
   - `platform-spec.md`
   - `requirements.md`

2. **如果未找到规格文档**
   - 🔴 **立即停止执行**
   - 提示用户：
     - "未找到项目规格文档"
     - "请提供项目规格文档，或者告诉我规格文档的路径"
     - "规格文档应包含：平台目录结构、页面布局规范、交互设计规范、样式规范等"
   - 等待用户提交规格文档或提供路径

3. **找到规格文档后**
   - 读取并确认文档内容
   - 向用户展示找到的规格文档路径
   - 请求用户确认是否使用此文档

### Phase 2: 详细分析规格文档

**深度分析规格文档内容：**

1. **提取关键信息**
   - 平台目录结构要求
   - 页面列表和布局规范
   - 组件和模块划分
   - 交互和样式要求
   - 技术栈要求
   - 测试要求

2. **识别模糊点**
   - 标记文档中不明确的描述
   - 识别需要用户确认的细节
   - 列出所有需要澄清的问题

3. **与用户确认模糊点**
   - 逐个询问模糊点
   - 记录用户的确认结果
   - 更新理解文档

### Phase 3: 生成详细 Task

**基于规格文档生成可执行的详细 task：**

1. **Task 拆分原则**
   - 每个 task 必须在 1-3 分钟内可执行完
   - 每个 task 必须是独立的、可验证的
   - 每个 task 必须有明确的完成标准
   - 按照依赖关系排序 task

2. **Task 结构**
   每个 task 必须包含：
   ```
   ### Task N: [任务名称]
   - 目标：[清晰的任务目标]
   - 文件：[精确的文件路径]
   - 前置任务：[依赖的 task 编号]
   - 执行步骤：
     Step 1: [具体操作，包含完整代码]
     Step 2: [运行测试的命令和预期输出]
     Step 3: [验证方法]
   - 完成标准：[明确的验收条件]
   - 预计耗时：[1-3 分钟]
   ```

3. **Task 分类**
   - 项目初始化类 task
   - 基础架构类 task
   - 页面组件类 task
   - 样式实现类 task
   - 交互逻辑类 task
   - 测试编写类 task
   - 集成验证类 task

4. **生成 task 清单**
   - 将 task 保存到 `docs/tasks/platform-build-tasks.md`
   - 向用户展示 task 清单概览
   - **等待用户批准 task 清单**

### Phase 4: 严格执行 Task

**按照批准的 task 清单逐个执行：**

1. **逐个任务执行**
   - 严格按照 task 顺序执行
   - 不要跳过任何 task
   - 不要合并多个 task
   - 每个 task 完成后立即验证

2. **Task 执行流程**
   - 读取当前 task 的详细步骤
   - 执行 Step 1: 编写代码/测试
   - 执行 Step 2: 运行验证命令
   - 执行 Step 3: 确认完成标准
   - 如果验证失败，立即修复
   - 验证通过后标记 task 完成

3. **进度跟踪**
   - 维护 task 执行进度清单
   - 每完成 5 个 task 向用户汇报一次
   - 遇到计划外问题立即停止并询问用户

4. **禁止行为**
   - ❌ 不要添加 task 清单中没有的功能
   - ❌ 不要修改已批准的 task 顺序
   - ❌ 不要跳过测试步骤
   - ❌ 不要在 task 外写代码

### Phase 5: 对照检查与测试

**所有 task 完成后进行全面检查：**

1. **功能对照检查**
   - 逐条对照规格文档的功能要求
   - 检查每个功能是否实现
   - 检查功能行为是否符合规格
   - 记录检查结果

2. **样式对照检查**
   - 逐条对照规格文档的样式要求
   - 检查颜色、字体、间距等
   - 检查响应式布局
   - 检查动画效果
   - 记录检查结果

3. **运行测试**
   - 运行所有单元测试
   - 运行集成测试
   - 运行端到端测试（如果有）
   - 确保所有测试通过
   - 生成测试报告

4. **问题修复**
   - 如果检查发现问题，创建修复 task
   - 执行修复 task
   - 重新验证
   - 直到所有检查通过

### Phase 6: 交付与验收

1. **生成交付清单**
   - 已实现的功能清单
   - 测试覆盖率报告
   - 已知问题清单
   - 部署说明

2. **用户验收**
   - 向用户展示交付清单
   - 引导用户验收测试
   - 记录用户反馈
   - 修复验收发现的问题

3. **最终交付**
   - 提交代码到版本控制
   - 提供部署文档
   - 提供使用文档（如需要）

## 行为规则

1. **规格文档优先**
   - ✅ 必须先找到规格文档才能开始
   - ✅ 规格文档不明确时必须询问用户
   - ❌ 不要假设用户的偏好
   - ❌ 不要添加规格文档中没有的功能

2. **Task 驱动执行**
   - ✅ 必须生成详细的 task 清单
   - ✅ 必须等待用户批准 task 清单
   - ✅ 必须严格按照 task 执行
   - ❌ 不要在 task 外写代码
   - ❌ 不要跳过任何 task

3. **时间盒原则**
   - ✅ 每个 task 必须在 1-3 分钟内完成
   - ✅ 如果 task 超时，拆分成更小的 task
   - ✅ 保持 task 的原子性和可验证性

4. **验证驱动**
   - ✅ 每个 task 完成后必须立即验证
   - ✅ 所有 task 完成后必须对照规格检查
   - ✅ 必须运行所有测试
   - ❌ 不要跳过验证步骤

5. **透明度**
   - ✅ 定期向用户汇报进度
   - ✅ 遇到问题立即告知用户
   - ✅ 所有检查结果向用户公开

## 输出物

- 项目规格文档分析报告
- 详细 task 清单 (`docs/tasks/platform-build-tasks.md`)
- task 执行进度报告
- 功能对照检查清单
- 样式对照检查清单
- 测试报告
- 交付清单

## 注意事项

- 🔴 **没有规格文档绝对不能开始**
- 🔴 **用户批准 task 清单之前不能写代码**
- 🔴 **必须严格按照 task 执行，不要自作主张**
- 🔴 **每个 task 必须在 1-3 分钟内可完成**
- 🔴 **完成后必须对照规格文档检查功能和样式**
- 🔴 **必须运行测试并生成测试报告**
