# open-app 业务架构

## 1. 核心定位

**open-app** 是一个**企业通讯能力开放平台**，将 **XXX 通讯系统** 的核心能力（IM、Meeting、CloudBox 等）通过 API、事件、回调等方式开放给业务应用和个人应用。

### 2.4 四种开放模式对比

| 维度 | API 开放 | 事件开放 | 回调开放 | 机器人开放 |
|------|----------|----------|----------|------------|
| **调用方向** | 外部系统 → XXX 通讯系统 | XXX 通讯系统 → 外部系统 | XXX 通讯系统 → 外部系统 | 双向交互 |
| **触发方式** | 外部系统主动调用 | 事件自动推送 | 特定业务场景触发 | 消息/命令触发 |
| **实时性** | 实时 | 近实时 | 近实时 | 实时 |
| **典型场景** | 发送消息、创建会议 | 收到消息通知、会议开始通知 | 审批结果、表单提交 | 智能问答、业务办理 |
| **协议** | HTTP/REST | WebHook/消息队列 | WebHook/HTTPS | HTTP/REST + WebHook |
| **交互模式** | 请求 - 响应 | 单向推送 | 单向推送 | 多轮对话 |
| **鉴权方式** | AppKey + Token | 订阅验证 + 签名 | 签名验证 | Bot Token + 签名 |

### 7.1 名词解释
| 术语 | 解释 |
|------|------|
| **Provider** | 资源提供者，XXX 通讯系统各子系统 |
| **Consumer** | 资源使用者，在平台注册的应用 |
| **API 开放** | 外部系统主动调用 XXX 通讯系统 API |
| **事件开放** | 外部系统订阅 XXX 通讯系统事件推送 |
| **回调开放** | XXX 通讯系统 WebHook 回调到外部系统 |
| **机器人开放** | 外部系统创建机器人，实现双向消息交互 |
| **OAuth 2.0** | 开放授权标准，用户授权应用访问能力 |
| **WebHook** | HTTP 回调，事件触发时推送通知 |
| **Bot** | 机器人，可实现消息收发、命令交互、业务集成 |
