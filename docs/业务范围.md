# open-app 业务范围文档

## 1. 项目概述

### 1.1 项目名称
**open-app** - 企业应用开放平台

### 1.2 项目定位
open-app 是一个面向企业的综合应用开放平台，提供权限管理、身份认证、API 网关、管理控制台等核心子模块，支持企业应用的快速开发、部署和集成。

### 1.3 目标用户
| 用户类型 | 使用场景 | 核心需求 |
|----------|----------|----------|
| 企业开发者 | 开发企业应用 | 快速集成、统一权限、API 支持 |
| 系统管理员 | 管理应用和权限 | 配置管理、审计监控、合规报告 |
| 安全管理员 | 安全管理与审计 | 权限审批、日志审计、风险告警 |
| 终端用户 | 使用企业应用 | 单点登录、统一入口、便捷体验 |

## 2. 子模块业务范围

### 2.1 permission-app（权限管理子模块）

#### 业务目标
提供企业级权限管理服务，支持 RBAC 模型，满足企业应用的权限控制需求。

#### 核心功能
| 功能域 | 功能 | 说明 | 优先级 |
|--------|------|------|--------|
| **认证服务** | OAuth 2.0 认证 | 支持授权码模式、刷新令牌 | P0 |
| | JWT Token 管理 | 签发、验证、刷新 | P0 |
| | 飞书集成认证 | 与飞书平台对接 | P0 |
| **角色管理** | 角色 CRUD | 创建、查询、修改、删除角色 | P0 |
| | 角色分配 | 为用户分配/撤销角色 | P0 |
| | 内置角色保护 | super_admin、admin、viewer | P0 |
| **权限管理** | 权限定义 | 定义系统权限点 | P0 |
| | 权限分配 | 为角色授予/撤销权限 | P0 |
| | 权限检查 API | 服务端权限验证接口 | P0 |
| **权限粒度** | 菜单级权限 | 控制菜单/页面可见性 | P0 |
| | 功能级权限 | 控制按钮/操作可用性 | P0 |
| | 数据级权限 | 控制数据访问范围（部门/项目） | P1 |
| **审计日志** | 操作日志 | 记录敏感操作 | P1 |
| | 权限变更日志 | 记录权限分配/撤销 | P1 |
| | 审计查询 | 支持日志检索和导出 | P1 |

#### 边界
**包含**:
- 用户身份认证
- 角色和权限管理
- 权限检查服务
- 审计日志记录

**不包含**:
- 用户基础信息管理（由 identity-app 负责）
- 组织架构管理（由 identity-app 负责）
- 外部身份提供商集成（由 identity-app 负责）

#### 依赖关系
- **依赖**: identity-app（用户数据源）
- **被依赖**: 所有需要权限控制的子模块

### 2.2 identity-app（身份管理子模块）- 规划中

#### 业务目标
提供统一的身份管理服务，管理用户、组织架构和外部身份提供商集成。

#### 核心功能（规划）
| 功能域 | 功能 | 说明 | 优先级 |
|--------|------|------|--------|
| **用户管理** | 用户 CRUD | 用户信息的增删改查 | P0 |
| | 用户状态管理 | 启用/禁用/锁定用户 | P0 |
| | 用户批量操作 | 批量导入/导出 | P1 |
| **组织架构** | 部门管理 | 部门的增删改查 | P0 |
| | 组织树 | 树形结构展示和维护 | P0 |
| | 岗位管理 | 岗位定义和关联 | P1 |
| **外部集成** | LDAP/AD 集成 | 与企业 AD/LDAP 对接 | P1 |
| | SSO 单点登录 | SAML/OIDC 协议支持 | P1 |
| | 飞书同步 | 与飞书组织架构同步 | P0 |

#### 边界
**包含**:
- 用户全生命周期管理
- 组织架构管理
- 外部身份源集成

**不包含**:
- 权限管理（由 permission-app 负责）
- 认证服务（由 permission-app 负责）

---

### 2.3 gateway-app（API 网关子模块）- 规划中

#### 业务目标
提供统一的 API 网关服务，管理 API 路由、限流、安全和监控。

#### 核心功能（规划）
| 功能域 | 功能 | 说明 | 优先级 |
|--------|------|------|--------|
| **路由管理** | API 路由配置 | 定义路由规则 | P0 |
| | 请求转发 | 转发请求到后端服务 | P0 |
| | 负载均衡 | 多实例负载分发 | P1 |
| **安全保护** | 限流熔断 | 防止过载和雪崩 | P0 |
| | IP 白名单 | IP 访问控制 | P0 |
| | 请求签名 | 验证请求合法性 | P1 |
| **监控告警** | 访问统计 | API 调用统计 | P1 |
| | 异常告警 | 错误率/延迟告警 | P1 |

#### 边界
**包含**:
- API 路由和转发
- 请求限流和熔断
- API 访问监控

**不包含**:
- 业务逻辑处理（由各子模块负责）
- 用户认证（由 permission-app 负责）

---

### 2.4 admin-app（管理控制台子模块）- 规划中

#### 业务目标
提供统一的管理控制台，支持各子模块的配置、监控和管理。

#### 核心功能（规划）
| 功能域 | 功能 | 说明 | 优先级 |
|--------|------|------|--------|
| **仪表盘** | 系统概览 | 展示关键指标 | P0 |
| | 实时监控 | 系统状态监控 | P1 |
| **配置管理** | 子模块配置 | 配置各子模块参数 | P0 |
| | 系统设置 | 全局系统配置 | P0 |
| **用户管理** | 用户查询 | 查看用户信息 | P0 |
| | 角色分配 | 为用户分配角色 | P0 |
| **审计中心** | 日志查询 | 查询操作日志 | P1 |
| | 报表导出 | 生成审计报告 | P1 |

#### 边界
**包含**:
- 可视化配置界面
- 系统监控仪表盘
- 审计日志查询

**不包含**:
- 权限逻辑处理（由 permission-app 负责）
- 用户数据存储（由 identity-app 负责）

## 3. 业务流程

### 3.1 用户登录流程
```
┌─────────┐      ┌──────────────┐      ┌─────────────┐      ┌────────────┐
│  用户   │─────▶│ identity-app │─────▶│ permission- │─────▶│  admin-app │
│         │      │  (身份验证)   │      │    app      │      │  (跳转首页)│
└─────────┘      └──────────────┘      └─────────────┘      └────────────┘
                                            │
                                            ▼
                                     ┌─────────────┐
                                     │  飞书平台   │
                                     │ (OAuth 2.0) │
                                     └─────────────┘
```

### 3.2 权限检查流程
```
┌─────────┐      ┌──────────────┐      ┌─────────────┐
│  请求   │─────▶│ gateway-app  │─────▶│ permission- │
│         │      │  (路由转发)   │      │    app      │
└─────────┘      └──────────────┘      └─────────────┘
                                            │
                                  ┌─────────┴─────────┐
                                  ▼                   ▼
                           ┌─────────────┐    ┌─────────────┐
                           │  角色查询   │    │  权限检查   │
                           │ identity-app│    │  (返回结果) │
                           └─────────────┘    └─────────────┘
```

### 3.3 权限分配流程
```
┌─────────┐      ┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│ 管理员  │─────▶│  admin-app  │─────▶│ permission- │─────▶│ 审计日志   │
│         │      │  (操作界面)  │      │    app      │      │  (记录操作)│
└─────────┘      └─────────────┘      └─────────────┘      └─────────────┘
                                            │
                                  ┌─────────┴─────────┐
                                  ▼                   ▼
                           ┌─────────────┐    ┌─────────────┐
                           │  更新角色   │    │  更新权限   │
                           │  关联关系   │    │  分配记录   │
                           └─────────────┘    └─────────────┘
```

## 4. 系统边界

### 4.1 与飞书平台的集成边界
| 集成项 | 方向 | 说明 |
|--------|------|------|
| OAuth 2.0 认证 | 飞书 → open-app | 使用飞书账号登录 |
| 用户信息同步 | 飞书 → identity-app | 同步用户基础信息 |
| 组织架构同步 | 飞书 → identity-app | 同步部门结构 |
| 消息推送 | open-app → 飞书 | 发送通知消息 |
| 事件订阅 | 飞书 → open-app | 接收用户/应用事件 |

### 4.2 与外部系统的边界
| 外部系统 | 集成方式 | 说明 |
|----------|----------|------|
| LDAP/AD | LDAP 协议 | 企业目录服务集成 |
| SSO 系统 | SAML/OIDC | 单点登录集成 |
| 审计系统 | API/日志 | 审计日志导出 |
| 监控系统 | API/Webhook | 告警通知 |

## 5. 非功能性需求

### 5.1 性能要求
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 认证响应时间 | < 200ms | P95 |
| 权限检查响应时间 | < 50ms | P95 |
| 系统可用性 | ≥ 99.9% | 月度 |
| 并发用户支持 | ≥ 10,000 | 同时在线 |

### 5.2 安全要求
| 要求 | 说明 |
|------|------|
| 服务端验证 | 所有权限检查必须在服务端执行 |
| 审计日志 | 敏感操作必须记录审计日志 |
| 最小权限 | 遵循最小权限原则 |
| 凭证加密 | 敏感凭证必须加密存储 |
| IP 白名单 | 支持 IP 访问控制 |

### 5.3 合规要求
| 要求 | 说明 |
|------|------|
| 数据可追溯 | 权限变更必须有完整记录 |
| 日志保留 | 审计日志至少保留 180 天 |
| 权限审批 | 敏感权限变更需要审批流程 |
| 定期审计 | 支持定期权限审计报告 |

## 6. 版本规划

### v1.0.0 - MVP (2026-Q2)
**范围**: permission-app 核心功能
- 用户认证（OAuth 2.0 / JWT）
- 角色管理（CRUD + 分配）
- RBAC 核心逻辑
- 基础 API
- 飞书认证集成

**不包含**:
- 数据级权限
- 审计日志
- 管理控制台

### v1.1.0 - 增强 (2026-Q3)
**范围**: permission-app 增强 + admin-app 启动
- 数据级权限（部门/项目维度）
- 审计日志
- admin-app 基础功能

**不包含**:
- identity-app
- gateway-app

### v2.0.0 - 扩展 (2026-Q4)
**范围**: identity-app + gateway-app
- identity-app 核心功能
- gateway-app 核心功能
- 外部 IdP 集成

## 7. 关键假设和依赖

### 7.1 关键假设
1. 企业已有飞书平台作为基础协作平台
2. 企业需要统一的权限管理服务
3. 开发者需要快速的 API 集成能力

## 8. 风险和缓解措施

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| 权限模型设计缺陷 | 中 | 高 | 先完成 ADR，评审后再实现 |
| 飞书 API 限制 | 中 | 中 | 实现缓存策略，降低调用频率 |
| 性能瓶颈（权限检查频繁） | 中 | 中 | 设计缓存策略，压力测试 |
| 需求变更 | 中 | 高 | 严格 SDD 流程，变更需更新规范 |

## 9. 名词解释

| 术语 | 解释 |
|------|------|
| RBAC | Role-Based Access Control，基于角色的访问控制 |
| ABAC | Attribute-Based Access Control，基于属性的访问控制 |
| OAuth 2.0 | 开放授权标准，允许第三方应用有限访问用户账号 |
| JWT | JSON Web Token，用于安全传输信息的令牌格式 |
| SSO | Single Sign-On，单点登录 |
| IdP | Identity Provider，身份提供商 |
| API Gateway | API 网关，统一管理 API 请求的入口 |
| SDD | Specification-Driven Development，规范驱动开发 |
