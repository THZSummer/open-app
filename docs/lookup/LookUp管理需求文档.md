## 2. 功能概述

### 2.1 背景

LookUp管理是系统的基础数据管理模块，用于统一管理业务系统中各类枚举值和配置项。通过标准化的管理机制，实现业务常量的集中维护和动态配置。

### 2.2 目标

- 提供统一的分类和LookUp项管理界面
- 支持LookUp数据的增删改查和状态管理
- 实现LookUp数据的层级组织和路径管理
- 提供灵活的扩展属性支持业务个性化需求
- 确保LookUp数据在系统中的一致性和可追溯性
- 支持LookUp明细项的批量导入功能

## 3. 用户故事

### US-001: 用户管理分类
作为用户，我想要管理分类，以便将LookUp项按业务模块进行组织，方便后续维护和查找。

**验收标准：**
- 可以查看所有分类列表
- 可以按分类编码、分类名称、描述、状态筛选分类
- 可以新增、编辑、删除分类
- 可以切换分类的生效/失效状态
- 删除分类时提示级联删除其下所有LookUp项

### US-002: 用户管理LookUp项
作为用户，我想要在分类下管理LookUp项，以便维护业务系统中使用的枚举值和配置项。

**验收标准：**
- 可以进入分类查看该分类下的LookUp项列表
- 可以按项编码、项名称、状态筛选LookUp项
- 可以新增、编辑、删除LookUp项
- 可以切换LookUp项的生效/失效状态
- 可以在右侧面板查看LookUp项详情
- 可以在详情面板直接编辑LookUp项

### US-004: 用户批量导入LookUp项
作为用户，我想要通过Excel文件批量导入LookUp项，以便快速初始化或更新大量字典数据，避免逐条新增的低效操作。

**验收标准：**
- 可以下载Excel导入模板
- 可以上传填写好的Excel文件进行批量导入
- 导入完成后显示成功/失败数量和失败原因
- 失败记录不影响成功记录的导入

## 4. 功能需求详情

### 4.1 分类管理模块

#### FR-001: 分类列表查询
**描述:** 分页查询分类列表，支持多条件筛选

**输入:**
- 类别编码（classifyCode，模糊匹配）
- 类别名称（classifyName，模糊匹配）
- 描述（classifyDesc，模糊匹配）
- 状态（status，1-有效, 0-失效, 空-全部）
- 页码（pageNum，默认1）
- 每页条数（pageSize，默认10）

**输出:**
- 分类列表（包含序号、编码、名称、描述、LookUp项数量、状态、创建人、创建时间）
- 分页信息（总条数、总页数、当前页）

**验证规则:**
- 分页参数（pageNum/pageSize）需为正整数
- LookUp项数量实时计算该分类下的所有LookUp项总数
- 列表按创建时间倒序排列

#### FR-002: 新增分类
**描述:** 创建新的分类

**输入:**
- 类别编码（classifyCode，必填，唯一）
- 类别名称（classifyName，必填）
- 路径（path，可选，用于层级归类）
- 描述（classifyDesc，可选）

**输出:**
- 创建成功的分类信息
- 创建时间、创建人自动填充

**验证规则:**
- 类别编码（classifyCode）：必填，长度1-100字符，只能包含字母、数字、下划线，全局唯一
- 类别名称（classifyName）：必填，长度1-100字符
- 路径（path）：可选，长度0-100字符
- 同一路径（path）下类别编码（classifyCode）必须唯一

#### FR-003: 编辑分类
**描述:** 修改现有分类的信息

**输入:**
- 分类ID（classifyId，必填）
- 类别名称（classifyName，必填）
- 路径（path，可选）
- 描述（classifyDesc，可选）

**输出:**
- 更新后的分类信息
- 更新时间、更新人自动更新

**验证规则:**
- 分类ID（classifyId）必须存在
- 类别编码（classifyCode）不可修改
- 类别名称（classifyName）：必填，长度1-100字符
- 路径（path）：可选，长度0-100字符
- 描述（classifyDesc）：可选，长度0-4000字符

#### FR-004: 删除分类
**描述:** 物理删除分类及其下所有LookUp项

**输入:**
- 分类ID（classifyId，必填）

**输出:**
- 操作成功
- 记录删除操作日志

**验证规则:**
- 删除前需二次确认
- 物理删除该分类及其下所有LookUp项（DELETE FROM）

#### FR-005: 分类状态变更
**描述:** 切换分类的有效/失效状态

**输入:**
- 分类ID（classifyId，必填）
- 目标状态（status，1-有效, 0-失效）

**输出:**
- 操作成功
- 更新时间和更新人自动更新

**验证规则:**
- 状态变更立即生效
- 分类失效不影响其下LookUp项的状态

### 4.2 LookUp项管理模块

#### FR-006: LookUp项列表查询
**描述:** 分页查询指定分类下的LookUp项列表

**输入:**
- 分类ID（classifyId，必填）
- 项编码（itemCode，模糊匹配）
- 项名称（itemName，模糊匹配）
- 状态（status，1-有效, 0-失效, 空-全部）
- 页码（pageNum，默认1）
- 每页条数（pageSize，默认10）

**输出:**
- LookUp项列表（包含序号、编码、名称、项值、排序、状态、创建人、创建时间）
- 分页信息
- 分类信息（编码、名称）

**验证规则:**
- 分页参数（pageNum/pageSize）需为正整数
- 列表按排序（itemIndex）升序排列，无排序值的排在最后
- 分类信息展示在页面顶部

#### FR-007: 新增LookUp项
**描述:** 在指定分类下创建新的LookUp项

**输入:**
- 分类ID（classifyId，必填）
- 项编码（itemCode，必填）
- 项名称（itemName，必填）
- 项值（itemValue，可选）
- 排序（itemIndex，可选，数字）
- 描述（itemDesc，可选）
- 扩展属性1-6（itemAttr1~6，可选）

**输出:**
- 创建成功的LookUp项信息
- 状态默认为 1（有效）
- 创建时间、创建人自动填充

**验证规则:**
- 项编码（itemCode）：必填，长度1-100字符，同一分类下唯一
- 项名称（itemName）：必填，长度1-100字符
- 项值（itemValue）：可选，长度0-2000字符
- 排序（itemIndex）：可选，整数
- 描述（itemDesc）：可选，长度0-4000字符
- 扩展属性1-6（itemAttr1~6）：可选，text类型

#### FR-008: 编辑LookUp项
**描述:** 修改现有LookUp项的信息

**输入:**
- LookUp项ID（itemId，必填）
- 项名称（itemName，必填）
- 项值（itemValue，可选）
- 排序（itemIndex，可选）
- 描述（itemDesc，可选）
- 扩展属性1-6（itemAttr1~6，可选）

**输出:**
- 更新后的LookUp项信息
- 更新时间、更新人自动更新

**验证规则:**
- LookUp项ID（itemId）必须存在
- 项编码（itemCode）不可修改
- 分类ID（classifyId）不可修改
- 项名称（itemName）：必填，长度1-100字符
- 项值（itemValue）：可选，长度0-2000字符
- 排序（itemIndex）：可选，整数
- 描述（itemDesc）：可选，长度0-4000字符
- 扩展属性1-6（itemAttr1~6）：可选，text类型

#### FR-009: 删除LookUp项
**描述:** 物理删除指定的LookUp项

**输入:**
- LookUp项ID（itemId，必填）

**输出:**
- 操作成功
- 记录删除操作日志

**验证规则:**
- 删除前需二次确认
- 物理删除该LookUp项（DELETE FROM）

#### FR-010: LookUp项状态变更
**描述:** 切换LookUp项的有效/失效状态

**输入:**
- LookUp项ID（itemId，必填）
- 目标状态（status，1-有效, 0-失效）

**输出:**
- 操作成功
- 更新时间和更新人自动更新

**验证规则:**
- 状态变更立即生效

#### FR-011: LookUp项详情查看
**描述:** 查看LookUp项的完整信息

**输入:**
- LookUp项ID（itemId，必填）

**输出:**
- 完整LookUp项信息，包含：
  - 基本信息（所属分类、编码、名称、值、排序、状态、描述）
  - 扩展属性（属性1-6）
  - 操作记录（创建人、创建时间、修改人、修改时间）

**验证规则:**
- 在列表右侧滑出详情面板展示
- 面板包含编辑按钮，可切换到编辑模式

#### FR-014: 批量导入LookUp项
**描述:** 支持通过文件批量导入LookUp明细项

**输入:**
- 导入数据（data）：包含LookUp项列表，每条记录需指定所属分类编码

**Excel模板格式:**

| 列名 | 必填 | 说明 |
|------|------|------|
| 所属分类编码 | 是 | 对应已有分类的编码，如 USER_TYPE |
| 项编码 | 是 | LookUp项的唯一编码，同一分类下唯一 |
| 项名称 | 是 | LookUp项的名称 |
| 项值 | 否 | LookUp项的值 |
| 排序 | 否 | 数字，越小越靠前 |
| 描述 | 否 | 项的描述说明 |
| 扩展属性1 | 否 | 扩展字段 |
| 扩展属性2 | 否 | 扩展字段 |
| 扩展属性3 | 否 | 扩展字段 |
| 扩展属性4 | 否 | 扩展字段 |
| 扩展属性5 | 否 | 扩展字段 |
| 扩展属性6 | 否 | 扩展字段 |

**输出:**
- 导入结果统计（总数、成功数、失败数）
- 失败记录明细及错误原因

**验证规则:**
- LookUp项编码需符合编码规范
- 同一分类下LookUp项编码不能重复
- 编码重复的记录自动跳过，不影响其他记录导入
- 导入为批量操作，失败记录不影响成功记录导入（部分成功模式）
- 导入时自动填充创建人、创建时间
- 导入完成后记录操作日志
