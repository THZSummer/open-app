# 飞书开放平台权限架构设计

## 架构概述

飞书开放平台采用分层权限架构设计，确保 API 调用的安全性和数据的隔离性。整体架构分为五层：

```
┌─────────────────────────────────────────────────────────────────┐
│                    用户层 (End Users)                            │
│                        终端用户                                   │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 登录授权
┌──────────────────────────▼──────────────────────────────────────┐
│                  应用层 (Applications)                           │
│     ┌─────────────────┐           ┌─────────────────┐          │
│     │   自建应用       │           │   商店应用       │          │
│     │  (Custom App)   │           │  (Store App)    │          │
│     └────────┬────────┘           └────────┬────────┘          │
└──────────────┼─────────────────────────────┼───────────────────┘
               │                             │
               └─────────────┬───────────────┘
                             │
┌────────────────────────────▼───────────────────────────────────┐
│                  API 调用层 (API Layer)                          │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐                │
│   │ 通讯录 API │    │  消息 API │    │ 云文档 API │   ...        │
│   └──────────┘    └──────────┘    └──────────┘                │
└──────────┬─────────────────────────┬───────────────────────────┘
           │                         │
    ┌──────▼────────┐       ┌────────▼────────┐
    │  认证授权层    │       │    事件层        │
    │ Authentication│       │ Event Subscription│
    └───────────────┘       └─────────────────┘
```

## 一、认证授权层 (Authentication Layer)

### 1.1 访问凭证类型

飞书开放平台提供三种类型的访问凭证 (access_token)：

| 凭证类型 | 是否需要用户授权 | 有效期 | 用途 |
|---------|----------------|--------|------|
| `app_access_token` | 否 | 2 小时 | 应用身份调用 API |
| `tenant_access_token` | 否 | 2 小时 | 租户数据访问 |
| `user_access_token` | 是 | 可刷新 | 用户身份调用 API |

### 1.2 凭证获取流程

#### app_access_token (应用访问凭证)

```
┌──────────────┐
│  自建应用     │
│  app_id      │
│  app_secret  │
└──────┬───────┘
       │ POST /open-apis/auth/v3/app_access_token/internal
       │ Body: {app_id, app_secret}
       ▼
┌──────────────────────────────┐
│  返回：                       │
│  {                           │
│    "code": 0,                │
│    "app_access_token": "...",│
│    "expire": 7200            │
│  }                           │
└──────────────────────────────┘
```

**注意事项：**
- 最大有效期 2 小时
- 剩余有效期 < 30 分钟时，调用会返回新 token（旧 token 仍有效）
- 剩余有效期 ≥ 30 分钟时，返回原有 token

#### tenant_access_token (租户访问凭证)

```
┌──────────────┐
│  自建应用     │
│  app_id      │
│  app_secret  │
└──────┬───────┘
       │ POST /open-apis/auth/v3/tenant_access_token/internal
       │ Body: {app_id, app_secret}
       ▼
┌─────────────────────────────────┐
│  返回：                          │
│  {                              │
│    "code": 0,                   │
│    "tenant_access_token": "...",│
│    "expire": 7200               │
│  }                              │
└─────────────────────────────────┘
```

**刷新机制：**
- 剩余有效期 < 30 分钟：返回新 token（同时存在两个有效 token）
- 剩余有效期 ≥ 30 分钟：返回原有 token

#### user_access_token (用户访问凭证)

```
┌────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  用户   │───▶│ 授权页面     │───▶│ 授权码 code │───▶│ user_       │
│  登录   │    │ /authen/v1  │    │ (5 分钟有效) │    │ access_token│
└────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                        │
                                                        ▼
                                                 ┌─────────────┐
                                                 │ refresh_    │
                                                 │ token       │
                                                 └─────────────┘
```

**OAuth 2.0 流程：**
1. 用户访问授权页面，同意授权
2. 获取授权码 `code`（有效期 5 分钟，仅限一次使用）
3. 使用 code 换取 `user_access_token` 和 `refresh_token`
4. 使用 `refresh_token` 刷新 `user_access_token`

```
                    需要调用 API 吗？
                         │
           ┌─────────────┴─────────────┐
           │                          │
```

## 二、应用层 (Application Layer)

### 2.1 应用类型

| 应用类型 | 描述 | 适用场景 |
|---------|------|---------|
| **自建应用 (Custom App)** | 企业自己开发的应用，仅供本企业使用 | 内部系统、企业定制化工具 |
| **商店应用 (Store App)** | 发布到飞书应用商店，可供多企业使用 | SaaS 产品、通用工具 |

### 2.2 应用权限体系

```
┌─────────────────────────────────────────────────────────┐
│                    应用权限                              │
│                                                         │
│  ┌─────────────────┐     ┌─────────────────┐           │
│  │   API 权限       │     │  应用数据权限    │           │
│  │   (Scope)       │     │  (Data Scope)   │           │
│  └────────┬────────┘     └────────┬────────┘           │
│           │                       │                     │
│           ▼                       ▼                     │
│  ┌─────────────────────────────────────────┐           │
│  │           企业管理员授权                  │           │
│  └─────────────────────────────────────────┘           │
└─────────────────────────────────────────────────────────┘
```

### 2.3 API 权限 (Scope)

**权限分类：**

| 权限类型 | 说明 | 示例 |
|---------|------|------|
| 通讯录权限 | 读取/写入成员、部门信息 | `contact:user:readonly` |
| 消息权限 | 发送/读取消息 | `im:message:send` |
| 云文档权限 | 访问云文档 | `drive:file:readonly` |
| 日历权限 | 访问日历事件 | `calendar:calendar:readonly` |
| 邮箱权限 | 访问邮箱（内测） | `mail:mailbox:readonly` |

**权限获取流程：**
1. 在飞书开放平台为应用配置所需 API 权限
2. 企业管理员审核授权
3. 应用获得对应 API 的调用权限

### 2.4 应用数据权限

**自建应用数据权限：**

| 业务范围 | 权限配置方式 |
|---------|------------|
| 通讯录 | 配置可见范围（指定部门/成员） |
| 组织架构 | 配置可见范围 |
| 群组 | 配置可见范围 |
| 机器人 | 配置可见范围 |
| Payroll | 需单独授权 |

**商店应用数据权限：**
- 安装时由企业管理员授权
- 可配置全员可见或指定范围

## 三、API 调用层 (API Layer)

### 3.1 API 调用流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     API 调用完整流程                             │
│                                                                 │
│  1. 准备阶段                                                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │
│  │ 创建应用     │─▶│ 配置权限    │─▶│ 设置 IP 白名单 │            │
│  └─────────────┘  └─────────────┘  └─────────────┘            │
│                                                                 │
│  2. 获取凭证                                                     │
│  ┌─────────────┐  ┌─────────────┐                              │
│  │ 获取 app_   │─▶│ 获取 tenant_│                              │
│  │ access_token│  │ access_token│                              │
│  └─────────────┘  └─────────────┘                              │
│                                                                 │
│  3. 调用 API                                                     │
│  ┌─────────────────────────────────────────┐                   │

│  ┌─────────────┐  ┌─────────────┐                              │

└─────────────────────────────────────────────────────────────────┘
```

### 3.2 API 请求规范

**请求格式：**
```http
POST /open-apis/auth/v3/tenant_access_token/internal HTTP/1.1
Host: open.feishu.cn
Content-Type: application/json; charset=utf-8

{
    "app_id": "cli_xxxxx",
    "app_secret": "xxxxx"
}
```

**响应格式：**
```json
{
    "code": 0,
    "msg": "ok",
    "tenant_access_token": "t-xxxxx",
    "expire": 7200
}
```

**调用 API：**
```http
GET /open-apis/contact/v3/users/{user_id} HTTP/1.1
Host: open.feishu.cn
Authorization: Bearer t-xxxxx
Content-Type: application/json; charset=utf-8
```

### 3.3 错误处理

| 错误码 | 说明 | 处理建议 |
|-------|------|---------|
| 0 | 成功 | - |
| 99991401 | IP 不在白名单 | 检查 IP 白名单配置 |
| 99991663 | 无效凭证 | 重新获取 token |
| 99991664 | 凭证过期 | 刷新 token |
| 99991665 | 权限不足 | 检查应用权限配置 |

```
│  ┌─────────────┐      ┌─────────────┐      ┌─────────────┐ │
```

### 4.2 订阅身份

| 订阅身份 | 使用凭证 | 数据范围 | 适用场景 |
|---------|---------|---------|---------|
| 应用身份 | tenant_access_token | 应用数据权限范围内 | 大多数业务事件 |
| 用户身份 | user_access_token | 用户可见数据范围 | 云文档、日历、邮箱 |

### 4.3 事件版本

| 版本 | 特点 | 推荐度 |
|-----|------|-------|
| v1.0 | 早期版本，结构简单 | ⭐⭐ |
| v2.0 | 结构更完善，推荐订阅 | ⭐⭐⭐⭐⭐ |

**v2.0 事件结构：**
```json
{
    "schema": "2.0",
    "header": {
        "event_id": "f7984f25108f8137722bb63cee927e66",
        "token": "xxxx",
        "create_time": "1603977298000000",
        "event_type": "contact.user_group.created_v3",
        "tenant_key": "xxxxxxx",
        "app_id": "cli_xxxxxxxx"
    },
    "event": {}
}
```

### 4.4 推送机制

**失败重试：**
- 第 1 次重试：15 秒后
- 第 2 次重试：5 分钟后
- 第 3 次重试：1 小时后
- 第 4 次重试：6 小时后
- 最多重试 4 次

**事件去重：**
- v1.0：通过 `uuid` 字段判断
- v2.0：通过 `event_id` 字段判断

**配置说明：**
- 仅支持 IPv4 地址
- 必须为公网 IP
- 对凭证获取接口不生效
- 默认不启用，启用后必须配置

### 5.2 凭证安全

| 凭证类型 | 存储建议 | 刷新策略 |
|---------|---------|---------|
| app_access_token | 服务端加密存储 | 过期前 30 分钟刷新 |
| tenant_access_token | 服务端加密存储 | 过期前 30 分钟刷新 |
| user_access_token | 加密存储 + 安全传输 | 使用 refresh_token 刷新 |
| refresh_token | 最高级别加密 | 妥善保管，泄露需重新授权 |

## 六、最佳实践

### 6.1 凭证管理

```python
# 推荐：token 缓存 + 自动刷新
class TokenManager:
    def __init__(self, app_id, app_secret):
        self.app_id = app_id
        self.app_secret = app_secret
        self.token_cache = {}
    
    def get_tenant_token(self):
        # 检查缓存
        if self._is_token_valid():
            return self.token_cache['token']
        
        # 获取新 token
        token = self._fetch_new_token()
        self.token_cache = {
            'token': token,
            'expire_time': time.time() + 7200
        }
        return token
    
    def _is_token_valid(self):
        # 提前 30 分钟刷新
        safe_margin = 30 * 60
        return (
            'token' in self.token_cache and
            self.token_cache.get('expire_time', 0) - time.time() > safe_margin
        )
```

### 6.2 权限最小化原则

1. **只申请需要的权限** - 避免过度授权
2. **定期审查权限** - 移除不再使用的权限
3. **分环境配置** - 开发/测试/生产环境使用不同应用

### 6.3 事件处理

```python
# 事件去重处理
processed_events = set()

def handle_event(event_data):
    event_id = event_data['header']['event_id']
    
    # 检查是否已处理
    if event_id in processed_events:
        return {"status": "duplicate"}
    
    # 处理事件
    process_business_logic(event_data)
    
    # 记录已处理
    processed_events.add(event_id)
    
    return {"status": "success"}
```
