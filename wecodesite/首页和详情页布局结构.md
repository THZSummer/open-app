# 首页与详情页布局结构文档

## 一、整体架构概览

### 1.1 布局类型划分

本项目采用双模式布局设计，根据路由路径区分：

| 布局模式 | 路由特征 | 顶部导航 | 应用信息栏 | 侧边栏 | 内容区 |
|---------|---------|---------|-----------|--------|-------|
| **首页布局** | `pathname === '/'` | ✅ | ❌ | ❌ | ✅ |
| **详情页布局** | `pathname !== '/'` | ✅ | ✅ | ✅ | ✅ |

### 1.2 布局层级结构

```
┌─────────────────────────────────────────────────────────────┐
│                        Header (64px)                        │
├─────────────────────────────────────────────────────────────┤
│                    AppInfoBar (50px) [仅详情页]              │
├──────────────┬──────────────────────────────────────────────┤
│              │                                              │
│   Sidebar    │              Content Area                    │
│   (220px)    │         (calc(100vh - header))              │
│  [仅详情页]   │                                              │
│              │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

### 3.2 布局计算公式

```javascript
// 内容区高度计算 (有侧边栏)
const contentAreaHeight = `calc(100vh - ${HEADER_HEIGHT}px - ${APP_INFO_BAR_HEIGHT}px - ${PADDING}px)`;
// 即: calc(100vh - 64px - 50px - 24px)

// 侧边栏高度
const sidebarMainHeight = `calc(100vh - ${HEADER_HEIGHT}px - ${APP_INFO_BAR_HEIGHT}px - ${PADDING}px)`;
```

### 3.3 AppInfoBar 组件详情

```
┌─────────────────────────────────────────────────────────────┐
```

### 3.5 Content Area 路由对应

| 路由路径 | 页面组件 | 说明 |
|---------|---------|-----|
| `/basic-info?appId=x` | BasicInfo | 凭证和基础信息 |

## 四、通用样式规范

### 4.1 尺寸常量

| 常量 | 值 | 说明 |
|-----|-----|-----|
| HEADER_HEIGHT | 64px | 顶部导航高度 |
| APP_INFO_BAR_HEIGHT | 50px | 应用信息栏高度 |
| SIDE_PADDING | 16px | 侧边栏内边距 |
| PADDING | 24px | 内容区内边距 |
| SIDEBAR_WIDTH | 220px | 侧边栏宽度 |

### 4.2 背景样式

| 区域 | 背景色 |
|-----|-------|
| Header | #fff |
| AppInfoBar | #fff |
| Sidebar | #fff + 右侧 1px #f0f0f0 边框 |
| Content | linear-gradient(180deg, #f7f9fc 0%, #f0f4f9 100%) |

### 4.3 溢出处理

- **Layout**: `overflow: hidden`
- **Sider**: `overflowY: auto`, `overflowX: hidden`
- **Content**: `overflowY: auto`, `overflowX: hidden`

---

## 五、响应式行为

当前实现为**桌面端优先**，未做响应式适配。滚动条出现在：
- 侧边栏内容超出时（垂直滚动）
- 内容区内容超出时（垂直滚动）

## 六、路由结构

```
/                           → AppList (首页)
/basic-info                 → BasicInfo (详情页)
/members                    → Members (详情页)
/capabilities               → Capabilities (详情页)
/capability-detail          → CapabilityDetail (详情页)
/api-management             → ApiManagement (详情页)
/events                     → Events (详情页)
/version-release            → VersionRelease (详情页)
```

所有详情页路由支持 `?appId=` 查询参数传递应用 ID。
