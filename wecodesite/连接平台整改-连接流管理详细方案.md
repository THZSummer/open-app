# 连接平台整改-连接流管理详细方案

> 本文档描述连接流管理页面的详细整改方案，包含状态扩展、表格列扩展、“更多”菜单、复制、部署、失效、恢复等能力。

---

## 一、页面现状

- 状态只有未部署、运行中、已停止，缺少已失效。
- 未部署文案需调整为待部署。
- 表格缺连接流 ID、创建者、更新人。
- 操作按钮直接暴露在表格中，没有“更多”菜单。
- 缺复制流、复制 ID、部署、失效、恢复。
- 缺部署版本选择弹窗。

---

## 二、整改目标

- 状态调整为：待部署、运行中、已停止、已失效。
- 表格补充连接流 ID、创建者、更新人。
- 操作列固定展示编辑、配置、更多。
- “更多”菜单按状态展示不同操作。
- 新增复制流、复制 ID、部署、失效、恢复能力。
- 部署时弹出版本选择弹窗，只展示已发布版本。

## 四、整改点与实现思路

### 4.1 状态枚举

```js
/**
 * 连接流生命周期状态枚举
 * 0=待部署，1=运行中，2=已停止，3=已失效
 */
export const FLOW_STATUS_MAP = {
  0: { text: '待部署', color: 'default' },
  1: { text: '运行中', color: 'green' },
  2: { text: '已停止', color: 'red' },
  3: { text: '已失效', color: 'red' },
};
```

### 4.2 表格列扩展

目标列：连接流 ID、中文名称、英文名称、中文描述、英文描述、创建者、更新人、创建时间、更新时间、状态、操作。

### 4.3 操作列改为“更多”菜单

```jsx
/**
 * 渲染连接流操作列
 * @param {Object} params - 参数对象
 * @param {Object} params.record - 当前行记录
 * @param {Function} params.handleEdit - 编辑回调
 * @param {Function} params.handleView - 配置回调
 * @param {Function} params.handleMoreAction - 更多菜单操作回调
 * @returns {JSX.Element}
 */
const renderFlowActions = (params) => {
  const { record, handleEdit, handleView, handleMoreAction } = params;
  return (
    <Space size="small">
      <Button type="link" size="small" onClick={() => handleEdit(record)}>编辑</Button>
      <Button type="link" size="small" onClick={() => handleView(record)}>配置</Button>
      <Dropdown
        trigger={['click']}
        menu={{
          items: getFlowMoreMenuItems(record.lifecycleStatus),
          onClick: ({ key }) => handleMoreAction({ record, action: key }),
        }}
      >
        <Button type="link" size="small">更多</Button>
      </Dropdown>
    </Space>
  );
};
```

### 4.4 更多菜单规则

| 状态 | 菜单条目 |
|------|----------|
| 待部署 | 复制流、复制 ID、部署、失效 |
| 运行中 | 复制流、复制 ID、部署、停止 |
| 已停止 | 复制流、复制 ID、部署、启动、失效 |
| 已失效 | 复制流、复制 ID、恢复、删除 |

```jsx
/**
 * 根据连接流状态获取更多菜单项
 * @param {number} status - 连接流状态
 * @returns {Array} antd 菜单项
 */
export const getFlowMoreMenuItems = (status) => {
  const commonItems = [
    { key: 'copy', label: '复制流' },
    { key: 'copyId', label: '复制ID' },
  ];
  const map = {
    0: [{ key: 'deploy', label: '部署' }, { key: 'invalidate', label: '失效', danger: true }],
    1: [{ key: 'deploy', label: '部署' }, { key: 'stop', label: '停止' }],
    2: [{ key: 'deploy', label: '部署' }, { key: 'start', label: '启动' }, { key: 'invalidate', label: '失效', danger: true }],
    3: [{ key: 'restore', label: '恢复' }, { key: 'delete', label: '删除', danger: true }],
  };
  return [...commonItems, ...(map[status] || [])];
};
```

### 4.5 操作分发

复制流、复制 ID、启动、恢复直接执行；停止、失效、删除走二次确认；部署打开部署弹窗。

```jsx
/**
 * 处理更多菜单操作
 * @param {Object} params - 参数对象
 * @param {Object} params.record - 当前连接流记录
 * @param {string} params.action - 操作类型
 */
const handleMoreAction = async (params) => {
  const { record, action } = params;
  if (action === 'copy') return handleCopyFlow(record);
  if (action === 'copyId') return handleCopyFlowId(record);
  if (action === 'deploy') return openDeployModal(record);
  if (action === 'start') return handleStartFlow(record);
  if (action === 'restore') return handleRestoreFlow(record);
  setActionModal({ visible: true, action, record, loading: false });
};
```

### 4.6 部署弹窗

```jsx
/**
 * 部署连接流弹窗
 * @param {Object} props - 组件参数
 * @param {boolean} props.open - 是否打开
 * @param {Array} props.versions - 可部署版本列表
 * @param {Function} props.onCancel - 取消回调
 * @param {Function} props.onConfirm - 确认部署回调
 * @returns {JSX.Element}
 */
const DeployFlowModal = ({ open, versions, onCancel, onConfirm }) => {
  const [selectedVersionId, setSelectedVersionId] = useState(null);
  return (
    <Modal title="选择部署版本" open={open} onCancel={onCancel} onOk={() => onConfirm(selectedVersionId)} okText="确认部署" width={520}>
      <Radio.Group value={selectedVersionId} onChange={(event) => setSelectedVersionId(event.target.value)}>
        <Space direction="vertical">
          {versions.map((version) => <Radio key={version.id} value={version.id}>v{version.versionNo}</Radio>)}
        </Space>
      </Radio.Group>
    </Modal>
  );
};
```
