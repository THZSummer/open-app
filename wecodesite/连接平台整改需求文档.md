# 连接平台整改需求文档（主文档）

> 本文档为连接平台整改总览文档。四个核心页面的详细实现方案已拆分到四个副文档中；审批中心和运行记录作为附属改造范围在本文档中补充说明。

---

## 一、整改背景

连接平台当前涉及连接器管理、接口配置、连接流管理、连接流编辑器等页面。现有实现与交互文档存在差异，主要集中在状态管理、版本体系、节点编排、Cookie 认证参数引用、审批流接入等方面。

本期整改仅涉及前端实现。后端接口未稳定前，前端按约定端点和字段先行落地，后续联调时再根据后端契约调整。

## 三、四个页面现状、目标与对比

### 3.1 连接器管理

#### 现状

- 表格只展示名称、类型、描述、创建时间、更新时间、操作等基础字段。
- 操作列固定为编辑、配置、删除。
- 无连接器状态字段展示。
- 无失效、恢复操作。
- 删除弹窗文案与新交互规范不一致。

#### 目标

- 表格补充连接器 ID、创建者、更新人、状态。
- 新增连接器状态：正常、已失效。
- 正常状态显示编辑、配置、失效。
- 已失效状态显示编辑、配置、恢复、删除。
- 失效和删除需要二次确认，恢复直接执行。

#### 对比

| 维度 | 当前实现 | 目标实现 | 差异 |
|------|----------|----------|------|
| 字段 | 缺 ID、创建者、更新人、状态 | 补齐目标字段 | 表格列扩展 |
| 状态 | 无状态展示 | 已失效 / 正常，`0=已失效`、`1=正常` | 新增状态模型 |
| 操作 | 编辑 / 配置 / 删除 | 按状态显示失效 / 恢复 / 删除 | 操作列重构 |
| 弹窗 | 删除弹窗文案旧 | 失效、删除统一确认 | 弹窗配置调整 |

详细方案见：[连接平台整改-连接器管理详细方案.md](./连接平台整改-连接器管理详细方案.md)

### 3.2 接口配置

#### 现状

- 页面是单份接口配置编辑模式，保存后直接生效。
- 无版本选择、版本状态和版本操作。
- 页面默认可编辑，无只读态。
- HTTP 方法包含 PATCH。
- 认证方式仅支持 SOA、APIG。
- Cookie、数字签名认证缺失。
- 入参、出参区域缺少按参数位置区分的 Tab 说明。
- object / array 参数缺少子参数校验。

#### 目标

- 页面顶部增加版本选择卡。
- 默认只读，草稿版本点击编辑后进入编辑态。
- 草稿支持保存、发布；已发布支持复制到草稿、失效；已失效支持复制到草稿、恢复、删除。
- HTTP 方法仅保留 GET / POST / PUT / DELETE。
- 增加 Cookie 和数字签名认证。
- 入参配置按 HTTP 请求头、HTTP 请求体、URL 参数区分 Tab。
- 出参配置按 HTTP 响应头、HTTP 响应体区分 Tab。
- 切换 Tab 时只展示当前参数位置的数据，保存时按参数位置合并提交。
- object / array 参数必须包含基础类型子参数，否则禁止保存。

#### 对比

| 维度 | 当前实现 | 目标实现 | 差异 |
|------|----------|----------|------|
| 页面形态 | 单配置编辑页 | 版本选择 + 只读/编辑切换 | 页面结构重组 |
| 版本 | 无 | 草稿 / 已发布 / 已失效 | 新增版本体系 |
| HTTP 方法 | 含 PATCH | 移除 PATCH | 协议收敛 |
| 认证 | SOA / APIG | 增加 Cookie / 数字签名 | 认证扩展 |
| 参数位置 | 入参、出参未明确按位置拆分 | 入参按请求头 / 请求体 / URL；出参按响应头 / 响应体拆分 | Tab 与数据结构补充 |
| 校验 | 无复杂类型校验 | object / array 必须含基础类型子参数 | 保存前校验 |

详细方案见：[连接平台整改-接口配置详细方案.md](./连接平台整改-接口配置详细方案.md)

### 3.3 连接流管理

#### 现状

- 状态仅包含未部署、运行中、已停止。
- 缺少已失效状态。
- 表格缺连接流 ID、创建者、更新人。
- 操作列直接展示启动、停止、删除等按钮，未收敛到“更多”菜单。
- 缺复制流、复制 ID、部署、失效、恢复操作。

#### 目标

- 状态调整为待部署、运行中、已停止、已失效。
- 表格补充连接流 ID、创建者、更新人。
- 操作列固定为编辑、配置、更多。
- “更多”菜单根据状态展示复制流、复制 ID、部署、启动、停止、失效、恢复、删除。
- 新增部署弹窗，选择已发布版本后部署。

#### 对比

| 维度 | 当前实现 | 目标实现 | 差异 |
|------|----------|----------|------|
| 状态 | 3 态 | 4 态，新增已失效 | 状态枚举扩展 |
| 字段 | 缺 ID、创建者、更新人 | 表格补齐 | 列扩展 |
| 操作 | 按钮直接暴露 | 编辑 / 配置 / 更多 | 操作入口收敛 |
| 部署 | 无 | 选择版本后部署 | 新增 DeployFlowModal |

详细方案见：[连接平台整改-连接流管理详细方案.md](./连接平台整改-连接流管理详细方案.md)

### 3.4 连接流编辑器

#### 现状

- 画布初始为空，用户通过左侧节点库拖拽节点。
- 节点类型较少，主要为 trigger、connector、exit，对应触发器、连接器、数据输出。
- 无连线“+”插入节点能力。
- 删除节点不会自动重连。
- 连接器节点无 Cookie 认证参数引用面板。
- 无错误处理、并行、文本、数据处理等新节点。
- 无版本子系统、版本详情抽屉、版本详情对比、调试抽屉。

#### 目标

- 默认初始化 HTTP 触发器节点和文本节点，并将文本节点文案设置为“结束”。
- 隐藏左侧节点库，仅允许通过连线“+”插入节点。
- 节点体系调整为：触发器节点、连接器节点、数据输出节点、错误处理节点、循环、并行节点、条件、文本节点、数据处理节点。
- 本期开放触发器、连接器、数据输出、错误处理、并行、文本、数据处理；循环和条件仅补充类型定义，本期不开放添加。
- 错误处理、并行节点通过连线“+”插入时生成节点组，不是只生成单个节点。
- 数据处理节点类似数据输出节点，节点配置弹窗中新增参数时支持选择函数处理。
- 删除中间节点时自动重连前后节点。
- 连接器节点支持 Cookie 认证参数从上游参数引用。
- 增加连接流版本子系统、详情抽屉、详情对比、调试抽屉。

#### 对比

| 维度 | 当前实现 | 目标实现 | 差异 |
|------|----------|----------|------|
| 初始化 | 空画布 | 默认触发器 + 文本节点（结束） | 默认数据生成 |
| 新增节点 | 左侧节点库拖拽 | 连线“+”插入 | 交互模式变更 |
| 节点类型 | 触发器 / 连接器 / 数据输出 | 增加错误处理、并行、文本、数据处理，循环和条件仅定义不开放 | 节点系统扩展 |
| 节点组 | 无 | 错误处理、并行按节点组插入 | 插入逻辑扩展 |
| 数据处理 | 无 | 类似数据输出，参数可配置函数处理 | 新增处理节点和配置能力 |
| Cookie 引用 | 无 | 引用上游参数生成 valueSource | 新增配置链路 |
| 版本 | 无 | 6 状态版本体系 | 新增版本管理 |
| 调试 / 对比 | 无 | 调试抽屉、详情对比 | 新增验证和差异查看能力 |

详细方案见：[连接平台整改-连接流编辑器详细方案.md](./连接平台整改-连接流编辑器详细方案.md)

## 四、附属改造范围

### 4.1 审批中心

#### 现状

- 审批中心已有我的待审、我发起的、全部、审批流程配置等基础 Tab。
- 审批流程配置已有流程名称、流程代码、审批节点等基础配置能力。
- 现有流程代码未覆盖连接流发布审批场景。
- 应用级连接流审批缺少应用 ID 配置入口。
- 编辑审批流程时，流程代码存在被修改的风险。

#### 目标

- 新增 `global_connector_flow`：全局连接流审批。
- 新增 `app_connector_flow`：应用级连接流审批。
- 当 `code = app_connector_flow` 时，显示并必填应用 ID。
- 编辑时审批流程 code 不允许修改。
- 其他 Tab、列表、详情、审批意见等交互保持现状。

#### 对比

| 维度 | 当前实现 | 目标实现 | 差异 |
|------|----------|----------|------|
| 流程代码 | 未包含连接流审批代码 | 增加全局连接流、应用级连接流审批代码 | 流程类型扩展 |
| 应用 ID | 无应用级连接流审批应用 ID | 应用级连接流审批显示并必填应用 ID | 表单字段补充 |
| 编辑限制 | 流程代码可能被修改 | 编辑时流程代码不可修改 | 表单状态调整 |
| 其他审批交互 | 保持现状 | 保持现状 | 不扩大范围 |

### 4.2 运行记录

#### 现状

- 左侧导航中尚未提供运行记录入口。
- 缺少连接流运行记录列表页，无法集中查看连接流执行结果。
- 缺少按连接流、执行状态、触发方式、执行时间筛选运行记录的能力。
- 缺少运行详情查看能力，无法定位触发参数、节点执行过程、错误信息和输出数据。
- 缺少运行记录与连接流、连接流版本之间的关联展示。

#### 目标

- 左侧导航新增“运行记录”菜单，并注册运行记录页面路由。
- 新增运行记录列表页，展示执行 ID、连接流名称、连接流版本、触发方式、执行状态、执行耗时、开始时间、结束时间等信息。
- 支持按连接流名称、执行状态、触发方式、时间范围进行组合筛选。
- 支持分页、重置筛选、刷新列表等基础列表交互。
- 新增运行详情抽屉，展示基本信息、触发参数、节点执行明细、错误信息、输出数据。
- 节点执行明细按执行顺序展示节点名称、节点类型、执行状态、耗时、输入、输出、错误信息，便于排查链路问题。
- 运行记录页面只做查看和排障，不提供重跑、编辑连接流、修改版本等写操作。

#### 改动方案

| 模块 | 改动内容 | 实现思路 |
|------|----------|----------|
| 导航入口 | 增加运行记录菜单 | 左侧导航新增菜单项，点击跳转运行记录页面 |
| 路由 | 注册运行记录页面路由 | 增加 `/connect/run-log` 对应页面入口 |
| 列表查询 | 增加运行记录查询能力 | 页面初始化和筛选变化时按条件请求运行记录列表 |
| 筛选区 | 增加名称、状态、触发方式、时间范围筛选 | 搜索条件统一维护，搜索时重置到第一页 |
| 表格区 | 展示运行记录核心字段 | 表格字段聚焦执行定位和排障，状态使用标签展示 |
| 详情抽屉 | 展示单次执行完整链路 | 点击详情后打开右侧抽屉，按区块展示基本信息、触发参数、节点明细和输出 |
| JSON 展示 | 展示触发参数、输入输出、错误上下文 | 使用格式化 JSON 展示，复杂结构保持层级可读 |
| 权限范围 | 只读查看 | 页面不提供修改连接流、修改版本、重跑等操作 |

#### 对比

| 维度 | 当前实现 | 目标实现 | 差异 |
|------|----------|----------|------|
| 导航入口 | 无运行记录入口 | 增加运行记录菜单 | 导航补充 |
| 页面路由 | 无 | 注册 `/connect/run-log` | 新增页面入口 |
| 列表能力 | 无 | 支持运行记录列表、分页、刷新 | 新增运行记录查询页 |
| 筛选能力 | 无 | 支持名称、状态、触发方式、时间范围筛选 | 新增组合筛选 |
| 详情能力 | 无 | 支持运行详情抽屉 | 新增排障详情 |
| 节点链路 | 无 | 展示节点执行明细、输入输出、错误信息 | 新增链路追踪视图 |
| 操作范围 | 无 | 仅查看，不提供写操作 | 明确只读边界 |

## 六、风险与说明

1. 后端接口契约未最终稳定，端点和字段以当前交互文档约定为准。
2. 连接流编辑器涉及节点系统和画布交互重构，影响范围较大。
3. Cookie 认证值来源只完成配置入口和 valueSource 保存，运行时执行链路依赖后端编排能力。
4. 运行记录页面以只读排障为主，列表与详情字段依赖后端运行日志契约。
5. 文档中的代码片段只用于说明实现思路，不代表完整代码改动。
