# 连接平台整改-连接器管理详细方案

> 本文档描述连接器管理页面的详细整改方案，包含整改思路、涉及文件、重点代码片段和验证点。

---

## 一、页面现状

- 表格缺少连接器 ID、创建者、更新人、状态。
- 操作列固定展示编辑、配置、删除，未按状态区分。
- 无失效、恢复操作。
- 删除按钮始终展示，不符合“已失效后才可删除”的目标。
- `disableConnector` 状态值需修正。
- 新建/编辑弹窗和二次确认弹窗宽度需要调整。

---

## 二、整改目标

- 增加状态枚举：`0=已失效`、`1=正常`。
- 表格补齐连接器 ID、创建者、更新人、状态。
- 正常状态展示：编辑、配置、失效。
- 已失效状态展示：编辑、配置、恢复、删除。
- 失效和删除走二次确认；恢复直接执行。
- 修复失效接口并新增恢复接口。

## 四、整改点与实现思路

### 4.1 状态枚举

集中维护连接器状态，列表展示和操作判断共用。

```js
/**
 * 连接器状态枚举
 * 用于连接器列表状态展示和操作列判断
 */
export const CONNECTOR_STATUS_MAP = {
  0: { text: '已失效', color: 'red' },
  1: { text: '正常', color: 'green' },
};
```

### 4.2 表格列扩展

目标列顺序：连接器 ID、中文名称、英文名称、类型、中文描述、英文描述、创建者、更新人、创建时间、更新时间、状态、操作。

```jsx
/**
 * 渲染连接器状态
 * @param {number} status - 连接器状态
 * @returns {JSX.Element}
 */
const renderConnectorStatus = (status) => {
  const statusInfo = CONNECTOR_STATUS_MAP[status] || { text: '-', color: 'default' };
  return <Tag color={statusInfo.color}>{statusInfo.text}</Tag>;
};
```

### 4.3 操作列按状态展示

编辑、配置保持原交互，只调整右侧状态相关操作。

```jsx
/**
 * 渲染连接器操作按钮
 * @param {Object} params - 参数对象
 * @param {Object} params.record - 当前行数据
 * @param {Function} params.handleEdit - 编辑回调
 * @param {Function} params.handleConfigClick - 配置回调
 * @param {Function} params.handleDisableClick - 失效回调
 * @param {Function} params.handleRestoreClick - 恢复回调
 * @param {Function} params.handleDeleteClick - 删除回调
 * @returns {JSX.Element}
 */
const renderConnectorActions = (params) => {
  const { record, handleEdit, handleConfigClick, handleDisableClick, handleRestoreClick, handleDeleteClick } = params;
  const isInvalid = record.status === 0;

  return (
    <Space size="small">
      {/* 编辑和配置保持原交互 */}
      <Button type="link" size="small" onClick={() => handleEdit(record)}>编辑</Button>
      <Button type="link" size="small" onClick={() => handleConfigClick(record)}>配置</Button>
      {isInvalid ? (
        <>
          <Button type="link" size="small" onClick={() => handleRestoreClick(record)}>恢复</Button>
          <Button type="link" size="small" danger onClick={() => handleDeleteClick(record)}>删除</Button>
        </>
      ) : (
        <Button type="link" size="small" danger onClick={() => handleDisableClick(record)}>失效</Button>
      )}
    </Space>
  );
};
```

### 4.4 失效、恢复、删除交互

失效和删除复用二次确认弹窗；恢复直接执行接口。

```jsx
/**
 * 打开失效确认弹窗
 * @param {Object} record - 当前连接器记录
 */
const handleDisableClick = (record) => {
  setActionModal({ visible: true, type: 'disable', record, loading: false });
};

/**
 * 恢复连接器
 * @param {Object} record - 当前连接器记录
 */
const handleRestoreClick = async (record) => {
  const result = await restoreConnector({ connectorId: record.id });
  if (result?.code === '200') {
    message.success('恢复成功');
    loadData();
    return;
  }
  message.error(result?.messageZh || '恢复失败');
};
```

### 4.5 thunk 修复与补充

```js
/**
 * 失效连接器
 * @param {Object} params - 参数对象
 * @param {string} params.connectorId - 连接器 ID
 * @returns {Promise<Object>} 接口响应
 */
export const disableConnector = async (params) => {
  const { connectorId } = params;
  return fetchApi(buildApiUrl(API_CONFIG.CONNECTORS.UPDATE, { connectorId }), {
    method: 'PUT',
    body: JSON.stringify({ status: 0 }),
  });
};

/**
 * 恢复连接器
 * @param {Object} params - 参数对象
 * @param {string} params.connectorId - 连接器 ID
 * @returns {Promise<Object>} 接口响应
 */
export const restoreConnector = async (params) => {
  const { connectorId } = params;
  return fetchApi(buildApiUrl(API_CONFIG.CONNECTORS.UPDATE, { connectorId }), {
    method: 'PUT',
    body: JSON.stringify({ status: 1 }),
  });
};
```

## 六、验证点

- 正常连接器仅展示编辑、配置、失效。
- 已失效连接器展示编辑、配置、恢复、删除。
- 失效后状态变为已失效。
- 恢复后状态变为正常。
- 删除仅在已失效状态可见。
- 表格新增字段正常展示。
