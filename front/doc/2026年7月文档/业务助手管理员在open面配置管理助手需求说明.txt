wecodesite
背景：现在每天有5~10个业务助手方，需要修改助手生产配置；修改uat配置进行联调验证。有些运营运维工作，支撑不及时、也不准确。业务体验不好。团队维护成本也高
需求价值目标：效率提升，释放运维人力；提升业务体验
需求范围：
1.业务中心和助手广场，基于APPID，将企业应用和业务助手关联
2.业务中心提供业务助手的管理入口：包括管理员角色和权限配置、管理页面菜单入口
3.业务中心装载（嵌入）助手广场的管理页面，包括助手基本信息管理、技能管理、首页配置
预期目标：
1.助手广场相关内容在wecodesite中暂不关注，wecodesite只关注如何嵌入助手广场相关页面
2.添加应用能力页面逻辑不变，仅页面数据可在marketadmin中配置
3.左侧添加的能力菜单入口，如果是嵌入的其他项目工程，能正常展示其他工程的页面内容
4.后续有其他的工程需要嵌入到wecodesite时，仅需要在marketadmin中配置对应的能力和应用地址，便可在wecodesite中展示，无需再修改wecodesite代码

market-web
背景：需要为应用提供‘能力’展示的配置能力，可以在页面配置应用锁提供能力信息（标题、描述、图标、示意图、排序、访问地址），配置完成后，在wecodesite中及时生效展示
需求价值目标：
1.提供应用能力的增删改查管理界面
2.支持配置能力的标题、描述、图标、示意图、排序、访问地址
功能清单：
1.能力列表展示——表格形式展示所有已配置的能力
2.新增能力——弹窗表单，填写6项配置字段
3.编辑能力——复用新增弹窗，回填已有数据
4.删除能力——二次确认后删除
5.图标上传——上传能力图标，支持预览和替换
6.示意图上传——上传能力示意图，支持预览和替换
页面设计：
1.页面分支顶部操作区域和列表区域两个部分
2.顶部操作区域：
2.1、页面标题：应用能力设置
2.2、页面说明：配置应用展示的能力信息，保存后即使生效
2.3、操作按钮：添加能能力
3.列表区域：
3.1、表格列定义
| 列名 | 说明 |
| 序号 | 显示排序数值 |
| 能力名称 | 展示图标 + 标题 + 描述 |
| 访问地址 | 超长省略，鼠标hover时在气泡框中展示完整地址 |
| 示意图 | 缩略图展示，未上传展示“——” |
| 操作账号 | 记录最后一次操作人员账号 |
| 更新时间| 记录最后一次操作时间 |
| 操作 | 编辑和删除，删除需要二次确认 |
4.字段校验
4.1、标题不能为空，长度为2~30个字符
4.2、 描述不能为空，长度5~200个字符
4.3、图标必传，尺寸40*40，不超过200K
4.4、示意图非必填，尺寸520*288，不超过500K
4.5、顺序必须为正整数，且大于1
4.6、访问地址不能为空，必须是合法http或https地址，长度不超过1000个字符
