## 1 需求价值和概述

审批中心已有我的待审、我发起的、全部、审批流程配置等基础能力，但流程代码未覆盖连接流发布审批场景，应用级连接流审批缺少应用 ID 配置，编辑审批流程时流程代码可能被修改。本次整改仅对审批流程配置做局部增强。

价值：支持全局和应用级连接流发布审批；防止流程代码被误改；应用级审批可绑定具体应用。

## 2 上下文分析

| 角色 | 关注点 |
|---|---|
| 平台管理员 | 配置审批流程和审批节点 |
| 审批人 | 处理连接流发布审批 |
| 连接流发布人 | 提交连接流发布审批 |

审批中心服务连接流版本发布流程。全局连接流审批适用于平台级发布，应用级连接流审批需要绑定应用 ID。

## 5 系统用例分析

| 角色 | UseCase | 说明 | 是否细化 |
|---|---|---|---|
| 管理员 | 配置连接流审批流程 | 新增或编辑审批流程配置 | 是 |
| 发布人 | 提交连接流发布审批 | 发布时进入匹配审批流程 | 是 |
| 审批人 | 审批连接流发布 | 在待审中处理审批任务 | 否 |

成功场景：管理员新增审批流程，选择连接流审批代码；若为应用级，填写应用 ID；保存成功后流程可用于连接流发布审批。  
扩展场景：编辑流程时 code 禁用；应用级未填写应用 ID 时阻止保存。

## 6 功能设计

### 6.1 业界方案实现

审批流程通常通过流程代码绑定业务场景，编辑后不允许修改 code，避免历史审批实例和新流程配置关系混乱。

### 6.2 功能实现整体设计方案

审批中心仅局部调整审批流程配置表单：扩展流程代码选项；根据 code 控制应用 ID 字段显隐和必填；编辑态禁用 code 字段。其他审批 Tab、列表、详情、意见等能力保持现状。

### 6.3 功能实现

| 流程代码 | 说明 | 应用 ID |
|---|---|---|
| `global_connector_flow` | 全局连接流审批 | 不展示 |
| `app_connector_flow` | 应用级连接流审批 | 展示并必填 |

| 模块 | 设计 |
|---|---|
| 流程代码选项 | 增加两个连接流审批代码 |
| 应用 ID 字段 | 当 code 为 `app_connector_flow` 时展示并必填 |
| 编辑态限制 | 编辑审批流程时 code 禁用 |
| 其他审批功能 | 保持现状 |

#### 6.3.1 接口设计

| URL | method | 功能 | 增删改查 | 鉴权 |
|---|---|---|---|---|
| `/approval/processes` | GET | 查询审批流程配置 | 查 | 是 |
| `/approval/processes` | POST | 新增审批流程 | 增 | 是 |
| `/approval/processes/{processId}` | PUT | 更新审批流程 | 改 | 是 |
| `/approval/processes/{processId}` | DELETE | 删除审批流程 | 删 | 是 |

#### 6.3.2 数据模型设计

| 字段 | 类型 | 说明 |
|---|---|---|
| processId | string | 审批流程 ID |
| name | string | 流程名称 |
| code | string | 流程代码 |
| appId | string | 应用 ID，仅应用级连接流审批需要 |
| nodes | array | 审批节点配置 |
| status | number | 流程状态 |

## 7 系统级非功能设计

### 7.1 FMEA 影响分析

| 风险 | 影响 | 措施 |
|---|---|---|
| 编辑时修改 code | 历史审批关联混乱 | 编辑态禁用 code |
| 应用级缺 appId | 审批路由不明确 | app_connector_flow 必填 appId |
| 流程代码重复 | 配置冲突 | 后端校验唯一性 |

### 7.2 安全影响分析

审批流程配置依赖后端鉴权；前端只做显示和必填校验，不替代权限控制。

### 7.3 兼容性

原有审批流程代码和审批列表交互保持不变；新增代码为增量扩展。

### 7.4 可运维

流程代码固定后便于排查审批路由；应用 ID 明确应用级审批范围。

### 7.5 资料

更新审批流程配置说明，补充连接流审批流程代码含义。
