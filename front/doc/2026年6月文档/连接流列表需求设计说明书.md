## 1 需求价值和概述

连接流列表当前包含运行中、已停止状态，缺少已失效状态；新建连接流默认应为已停止状态；表格字段缺少连接流 ID、创建者、更新人；操作按钮直接暴露在表格中，缺少统一的更多菜单；同时，"启动"操作既切换状态又携带版本选择，语义混合，导致版本变更必须通过先停止再启动来完成。现有列表难以完整表达连接流生命周期，也不便于用户定位、复制、启动和切换部署版本。

本次整改需要将连接流状态调整为已停止、运行中、已失效三种；补齐资产识别字段；将操作列收敛为编辑、配置、更多；在更多菜单中按状态展示可用操作；将"启动"拆分为两个独立动作——「启动」仅切换状态（已停止 → 运行中），「部署」仅修改连接流与已发布版本的绑定关系且不影响状态；运行中状态新增"部署"入口以支持热切换部署版本；补充复制流和复制 ID 能力。

价值：

| 价值点 | 说明 |
|---|---|
| 资产识别 | 展示连接流 ID、创建者、更新人，便于排查和管理 |
| 生命周期管理 | 支持已停止、运行中、已失效三态展示和操作 |
| 操作收敛 | 操作列固定展示编辑、配置、更多，减少列表横向按钮堆叠 |
| 动作语义清晰 | 启动只影响状态，部署只影响版本绑定，避免语义耦合 |
| 部署灵活性 | 运行中可直接切换部署版本，不需要停止再启动 |
| 启动准确性 | 启动前必须存在已发布的部署版本，避免空运行 |
| 复用效率 | 支持复制流，减少重复配置成本 |
| 运维定位 | 支持复制 ID，便于日志排查和接口联调 |

## 2 上下文分析

连接流列表本次整改来源于上一版本生命周期表达不完整和启动能力不足。上一版本列表能够承载基础查看、编辑和配置入口，但状态、字段和操作还停留在简单管理阶段，无法完整支撑连接流从创建、启动、运行、停止到失效删除的闭环管理。

| 背景类型 | 现有问题或增强原因 | 本次处理方向 |
|---|---|---|
| 上个版本功能缺陷 | 状态缺少已失效，且新建连接流默认状态不符合实际生命周期 | 状态调整为已停止、运行中、已失效三种，新建和复制后默认为已停止 |
| 上个版本功能缺陷 | 表格缺少连接流 ID、创建者、更新人，排查问题和审计变更时定位成本较高 | 补充资产识别和审计字段 |
| 上个版本功能缺陷 | 操作按钮直接铺在表格中，状态差异越多越难控制可见性，也容易暴露不应执行的操作 | 操作列收敛为编辑、配置、更多，并按状态生成更多菜单 |
| 上个版本功能缺陷 | "启动"既切换状态又选择版本，语义耦合；运行中切换版本必须先停止再启动，使用成本较高 | 将启动拆分为「启动」和「部署」两个独立动作，分别承担状态切换与版本绑定 |
| 上个版本功能缺陷 | 启动缺少前置依赖校验，可能在未部署版本的情况下被切换至运行中 | 启动要求连接流必须已存在部署版本，否则阻断 |
| 现有能力增强 | 用户需要快速复用连接流配置，并在排障时复制连接流 ID | 增加复制流和复制 ID 能力 |

因此，本次连接流列表不是单纯增加按钮，而是补齐连接流生命周期管理能力，降低启动误操作和排障成本，并让不同状态下的操作入口更清晰。

|---|---|---|

| 操作列 | 操作按钮直接暴露在表格中 | 按钮过多时影响列表展示，也不便按状态收敛操作 |

| 启动能力 | "启动"动作同时承担状态切换与版本选择 | 语义混合，运行中切换版本必须先停止再启动；缺乏针对版本绑定关系的独立入口 |

## 6 系统用例分析

连接流列表整改主要包含连接流资产查看、更多菜单操作、启动状态切换、部署版本选择和复制能力验证五个用例。

连接流资产查看：开发人员和测试人员需要验证列表是否展示连接流 ID、名称、描述、创建者、更新人和状态。
更多菜单操作：开发人员和测试人员需要验证不同状态下更多菜单条目是否正确，启动与部署是否同时存在于已停止状态，部署是否同样存在于运行中状态，停止、失效、删除是否需要二次确认。
启动状态切换：开发人员和测试人员需要验证启动操作仅切换状态（已停止 → 运行中），且在连接流未存在部署版本时阻断并提示。
部署版本选择：开发人员和测试人员需要验证部署操作是否打开"选择部署版本"弹窗，仅展示已发布版本，已部署连接流默认选中当前部署版本；确认部署后连接流状态保持不变，仅"当前部署版本"字段被更新。
复制能力验证：开发人员和测试人员需要验证复制流成功后刷新列表，复制 ID 成功后展示成功提示。

成功场景：用户进入连接流列表，表格展示完整字段和状态标签；操作列固定展示编辑、配置、更多；用户点击更多菜单后，系统按状态展示可用操作；已停止连接流点击启动后直接切换为运行中（前置已部署版本）；已停止或运行中连接流点击部署后打开"选择部署版本"弹窗；操作成功后刷新列表。
扩展场景：未部署版本时启动操作阻断并提示"当前连接流尚未部署版本，请先部署后再启动"；无已发布版本时部署弹窗提示"暂无已发布的版本"并禁用确认按钮；停止、失效、删除需要二次确认；复制 ID 失败时提示失败；操作失败时保留当前页面状态并提示错误。

## 7 功能设计

### 7.1 业界方案实现

流程类资源列表通常使用状态标签表达生命周期。高频操作前置展示，低频操作和危险操作收敛到更多菜单中，并根据资源状态控制菜单项可见性。"启动/停止"用于切换运行状态；"部署/发布"用于建立资源与具体版本的绑定关系；二者解耦后，运行时切换版本不需要先停后启。

### 7.2 功能实现整体设计方案

表格展示完整资产字段；操作列固定为编辑、配置、更多；更多菜单根据连接流状态生成；启动操作仅切换状态，并在前置依赖（已存在部署版本）满足时执行；部署操作打开版本选择弹窗，仅更新连接流与已发布版本的绑定关系；复制、复制 ID、恢复直接执行；停止、失效、删除进入二次确认；所有操作成功后刷新列表。

整体操作流程如下：

```mermaid
flowchart TD
    A[进入连接流列表] --> B[查看连接流资产字段和状态]
    B --> C{选择操作入口}
    C --> D[编辑]
    C --> E[配置]
    C --> F[更多菜单]
    F --> G[按状态生成可用菜单项]
    G --> H{操作类型}
    H --> I[复制流或复制 ID]
    H --> J[启动]
    H --> P1[部署]
    H --> K[恢复]
    H --> L[停止、失效或删除]
    J --> JV{是否存在部署版本}
    JV -->|否| JN[提示先部署再启动]
    JV -->|是| JY[仅切换状态为运行中]
    P1 --> M[打开部署版本选择弹窗]
    L --> N[二次确认]
    I --> O[执行操作并提示结果]
    K --> O
    M --> O
    N --> O
    JY --> O
    JN --> O
    O --> P{是否需要刷新列表}
    P -->|是| Q[刷新连接流列表]
    P -->|否| R[保留当前列表状态]
```

```mermaid
    已停止 --> 运行中: 启动（仅切换状态，要求已部署）
    运行中 --> 已停止: 停止（仅切换状态，部署版本绑定保持不变）
    已停止 --> 已停止: 部署（仅更新版本绑定，状态不变）
    运行中 --> 运行中: 部署（仅更新版本绑定，状态不变）
    已停止 --> 已失效: 失效
    已失效 --> 已停止: 恢复（保留原部署版本绑定）
    已失效 --> [*]: 删除
```

### 7.3 状态枚举设计

| 状态文案 | 颜色建议 | 说明 |
|---|---|---|
| 运行中 | 绿色 | 当前连接流已选择已发布版本并运行 |
| 已停止 | 红色 | 当前连接流未运行，新建和复制后默认进入该状态 |
| 已失效 | 红色 | 当前连接流已失效，可恢复或删除 |

未知状态展示默认占位，不展示危险操作。

### 7.6 更多菜单规则

| 规则 | 说明 |
|---|---|

| 操作 | 处理方式 | 成功后动作 | 失败后动作 |
|---|---|---|---|

| 失效 | 二次确认后调用状态变更接口 | 提示失效成功并刷新列表，状态变为已失效 | 保留页面状态并提示失败 |
| 恢复 | 调用状态变更接口 | 提示恢复成功并刷新列表，状态变为已停止 | 提示失败原因 |
| 删除 | 二次确认后调用删除接口 | 提示删除成功并刷新列表 | 保留页面状态并提示失败 |

## 11 系统级非功能设计

### 11.1 FMEA 影响分析

| 风险 | 影响 | 措施 |
|---|---|---|
| 无可部署版本 | 用户无法完成部署 | 部署弹窗提示"暂无已发布的版本"，禁用"确认部署"按钮 |
| 未部署直接启动 | 用户在未部署版本时启动连接流 | 启动前校验是否已部署，未部署时阻断并提示"请先部署后再启动" |
| 菜单状态错误 | 用户看到不应出现的操作 | 前端按状态控制菜单，后端继续校验状态 |
| 删除误操作 | 数据丢失 | 仅已失效可删除，并增加二次确认 |
| 停止误操作 | 运行中连接流被误停 | 停止前二次确认；停止不影响部署版本绑定，便于后续恢复运行 |
| 部署误切换 | 运行中误切换部署版本 | 弹窗默认选中当前部署版本，用户需主动选择其他版本；确认按钮文案明确 |
| 复制 ID 失败 | 运维定位受影响 | 提示失败，允许用户手动复制 |
| 操作失败 | 列表状态未按预期更新 | 保留当前页面状态，提示失败原因，成功后刷新列表 |
| 未知状态 | 展示和操作不确定 | 状态展示默认占位，隐藏高风险操作 |

### 11.2 安全影响分析

启动、部署、停止、失效、恢复、删除均依赖后端鉴权。前端按状态控制操作入口，后端仍需校验当前连接流状态、用户权限以及部署版本是否处于已发布状态。失效入口仅在已停止状态展示，删除入口仅在已失效状态展示，且停止、失效、删除前需要二次确认。

### 11.3 兼容性

历史"待部署"状态不再作为连接流有效生命周期状态展示。如存量数据仍返回待部署状态，前端应按未知状态处理，仅保留复制 ID 能力。旧数据缺少创建者、更新人时展示默认占位。历史的"启动并选择版本"接口需要按"启动"（仅状态变更）和"部署"（仅版本绑定）两个能力适配；存量已经处于运行中的连接流必定带有当前部署版本字段，新接口对其透明兼容。

### 11.4 可运维

复制 ID 便于日志排查和接口联调；启动、部署、状态变更和删除操作成功后刷新列表；失败时展示明确错误提示，便于用户判断是否需要重试或联系管理员。启动与部署的拆分使运维可以独立审计"状态变更日志"与"版本绑定变更日志"。

### 11.5 资料

需要更新连接流列表操作说明，补充状态含义、更多菜单规则、启动前置依赖、部署版本选择、复制流、复制 ID 和危险操作确认说明。
