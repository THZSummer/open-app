## 1 需求价值和概述

连接流列表当前仅包含未部署、运行中、已停止三种状态，缺少已失效状态；表格字段缺少连接流 ID、创建者、更新人；操作按钮直接暴露在表格中，缺少统一的更多菜单；部署时缺少版本选择入口。现有列表难以完整表达连接流生命周期，也不便于用户定位、复制和部署连接流。

本次整改需要将连接流状态调整为待部署、运行中、已停止、已失效四种；补齐资产识别字段；将操作列收敛为编辑、配置、更多；在更多菜单中按状态展示可用操作；部署时弹出版本选择弹窗，只展示已发布版本；补充复制流和复制 ID 能力。

价值：

| 价值点 | 说明 |
|---|---|
| 资产识别 | 展示连接流 ID、创建者、更新人，便于排查和管理 |
| 生命周期管理 | 支持待部署、运行中、已停止、已失效四态展示和操作 |
| 操作收敛 | 操作列固定展示编辑、配置、更多，减少列表横向按钮堆叠 |
| 部署准确性 | 部署前选择已发布版本，避免部署版本不明确 |
| 复用效率 | 支持复制流，减少重复配置成本 |
| 运维定位 | 支持复制 ID，便于日志排查和接口联调 |

## 2 上下文分析

| 角色 | 关注点 |
|---|---|
| 连接流开发人员 | 编辑、配置、复制连接流，复用已有流配置 |
| 平台管理员 | 部署、启动、停止、失效、恢复、删除连接流 |
| 运维人员 | 查看连接流状态，复制连接流 ID 定位问题 |

连接流列表是连接平台中的配置、管理和部署入口。“编辑”用于维护连接流基础信息；“配置”进入连接流编辑器；“部署”依赖已发布版本；状态变更会影响连接流是否可以运行、恢复或删除。

## 3 页面现状

| 现状项 | 当前表现 | 问题 |
|---|---|---|
| 状态枚举 | 只有未部署、运行中、已停止 | 缺少已失效，且未部署文案需要调整为待部署 |
| 表格字段 | 未展示连接流 ID、创建者、更新人 | 资产定位和审计信息不足 |
| 操作列 | 操作按钮直接暴露在表格中 | 按钮过多时影响列表展示，也不便按状态收敛操作 |
| 复制能力 | 缺少复制流、复制 ID | 复用配置和定位问题不方便 |
| 部署能力 | 缺少部署版本选择弹窗 | 用户无法明确选择要部署的已发布版本 |
| 生命周期操作 | 缺少失效、恢复等能力 | 连接流无法形成完整生命周期闭环 |

## 6 系统用例分析

| 角色 | UseCase | 说明 | 是否细化 |
|---|---|---|---|
| 平台管理员 | 管理生命周期 | 部署、启动、停止、失效、恢复、删除连接流 | 是 |
| 开发人员 | 复用连接流 | 复制已有连接流生成新流 | 是 |

成功场景：用户进入连接流列表，表格展示完整字段和状态标签；操作列固定展示编辑、配置、更多；用户点击更多菜单后，系统按状态展示可用操作；部署操作打开版本选择弹窗；操作成功后刷新列表。  
扩展场景：无已发布版本时提示暂无可部署版本；停止、失效、删除需要二次确认；复制 ID 失败时提示失败；接口失败时保留当前页面状态并提示错误。

## 7 功能设计

### 7.1 业界方案实现

流程类资源列表通常使用状态标签表达生命周期。高频操作前置展示，低频操作和危险操作收敛到更多菜单中，并根据资源状态控制菜单项可见性。部署类操作通常需要先选择目标版本，避免系统默认版本与用户预期不一致。

### 7.2 功能实现整体设计方案

表格展示完整资产字段；操作列固定为编辑、配置、更多；更多菜单根据连接流状态生成；部署操作打开版本选择弹窗；复制、复制 ID、启动、恢复直接执行；停止、失效、删除进入二次确认；所有操作成功后刷新列表。

### 7.3 状态枚举设计

| 状态值 | 状态文案 | 颜色建议 | 说明 |
|---|---|---|---|
| 0 | 待部署 | 默认色 | 原未部署状态统一展示为待部署 |
| 1 | 运行中 | 绿色 | 当前连接流已部署并运行 |
| 2 | 已停止 | 红色 | 当前连接流已停止运行 |
| 3 | 已失效 | 红色 | 当前连接流已失效，可恢复或删除 |

未知状态展示默认占位，不展示危险操作。

### 7.4 表格列设计

目标列顺序：

| 序号 | 字段 | 展示名称 | 说明 |
|---|---|---|---|
| 1 | id | 连接流 ID | 连接流唯一标识 |
| 2 | nameZh | 中文名称 | 连接流中文名称 |
| 3 | nameEn | 英文名称 | 连接流英文名称 |
| 4 | descriptionZh | 中文描述 | 连接流中文描述 |
| 5 | descriptionEn | 英文描述 | 连接流英文描述 |
| 6 | creator | 创建者 | 创建连接流的用户 |
| 7 | updater | 更新人 | 最近一次更新连接流的用户 |
| 8 | createTime | 创建时间 | 连接流创建时间 |
| 9 | updateTime | 更新时间 | 连接流更新时间 |
| 10 | lifecycleStatus | 状态 | 以状态标签展示 |
| 11 | action | 操作 | 固定展示编辑、配置、更多 |

### 7.5 操作列设计

操作列固定展示三个入口：

| 操作 | 展示方式 | 行为 |
|---|---|---|
| 编辑 | 按钮 | 打开连接流基础信息编辑入口 |
| 配置 | 按钮 | 进入连接流配置页面 |
| 更多 | 下拉菜单 | 按状态展示当前可用操作 |

编辑和配置保持原有交互，不受状态差异影响。更多菜单承载复制、部署、启动、停止、失效、恢复、删除等操作。

### 7.6 更多菜单规则

| 状态 | 菜单条目 | 说明 |
|---|---|---|
| 待部署 | 复制流、复制 ID、部署、失效 | 待部署流可复制、部署或置为失效 |
| 运行中 | 复制流、复制 ID、部署、停止 | 运行中的流可重新部署或停止 |
| 已停止 | 复制流、复制 ID、部署、启动、失效 | 已停止的流可启动、部署或置为失效 |
| 已失效 | 复制流、复制 ID、恢复、删除 | 已失效的流可恢复或删除 |
| 未知状态 | 复制 ID | 未知状态下仅保留低风险定位能力 |

菜单规则补充：

| 规则 | 说明 |
|---|---|
| 复制流 | 用于基于当前连接流生成副本 |
| 复制 ID | 将连接流 ID 写入剪贴板 |
| 部署 | 打开部署版本选择弹窗 |
| 启动 | 直接执行启动操作 |
| 停止 | 进入二次确认弹窗 |
| 失效 | 进入二次确认弹窗 |
| 恢复 | 直接执行恢复操作 |
| 删除 | 仅已失效状态展示，并进入二次确认弹窗 |

### 7.7 操作分发设计

| 操作 | 处理方式 | 成功后动作 | 失败后动作 |
|---|---|---|---|
| 复制流 | 调用复制流接口 | 提示复制成功并刷新列表 | 提示失败原因 |
| 复制 ID | 调用复制 ID 工具能力 | 提示复制成功 | 提示复制失败 |
| 部署 | 打开部署版本选择弹窗 | 确认部署成功后刷新列表 | 提示失败原因 |
| 启动 | 调用状态变更接口 | 提示启动成功并刷新列表 | 提示失败原因 |
| 停止 | 二次确认后调用状态变更接口 | 提示停止成功并刷新列表 | 保留页面状态并提示失败 |
| 失效 | 二次确认后调用状态变更接口 | 提示失效成功并刷新列表 | 保留页面状态并提示失败 |
| 恢复 | 调用状态变更接口 | 提示恢复成功并刷新列表 | 提示失败原因 |
| 删除 | 二次确认后调用删除接口 | 提示删除成功并刷新列表 | 保留页面状态并提示失败 |

### 7.8 部署版本选择弹窗设计

| 项目 | 设计 |
|---|---|
| 打开时机 | 点击更多菜单中的部署 |
| 弹窗标题 | 选择部署版本 |
| 数据来源 | 当前连接流的可部署版本列表 |
| 版本范围 | 只展示已发布版本 |
| 选择方式 | 单选 |
| 确认按钮 | 确认部署 |
| 取消按钮 | 关闭弹窗，不执行部署 |
| 无数据处理 | 提示暂无已发布版本，禁止确认部署 |
| 成功处理 | 提示部署成功并刷新列表 |
| 失败处理 | 提示部署失败原因，弹窗状态按实际交互保留或关闭 |

### 7.9 复制能力设计

| 能力 | 设计 |
|---|---|
| 复制流 | 调用复制接口生成新的连接流记录，成功后刷新列表 |
| 复制 ID | 复制当前行连接流 ID，成功后提示“复制成功” |
| 失败处理 | 复制流失败展示接口错误；复制 ID 失败提示用户手动复制 |

### 7.10 生命周期操作设计

| 操作 | 来源状态 | 目标状态 | 是否确认 | 说明 |
|---|---|---|---|---|
| 部署 | 待部署、运行中、已停止 | 运行中 | 否，需选择版本 | 确认版本后执行部署 |
| 启动 | 已停止 | 运行中 | 否 | 直接执行启动 |
| 停止 | 运行中 | 已停止 | 是 | 防止误停运行中连接流 |
| 失效 | 待部署、已停止 | 已失效 | 是 | 失效后可恢复或删除 |
| 恢复 | 已失效 | 待部署或后端指定状态 | 否 | 直接执行恢复，最终状态以后端返回为准 |
| 删除 | 已失效 | 删除记录 | 是 | 仅已失效状态允许删除 |

### 7.11 接口设计

| URL | method | 功能 | 增删改查 | 鉴权 | 说明 |
|---|---|---|---|---|---|
| `/flows` | GET | 查询连接流列表 | 查 | 是 | 返回列表字段和生命周期状态 |
| `/flows/{flowId}/copy` | POST | 复制连接流 | 增 | 是 | 基于当前连接流创建副本 |
| `/flows/{flowId}/deploy` | POST | 部署连接流版本 | 改 | 是 | 入参包含选择的版本标识 |
| `/flows/{flowId}/status` | PUT | 启动、停止、失效、恢复 | 改 | 是 | 根据目标状态更新连接流生命周期 |
| `/flows/{flowId}` | DELETE | 删除已失效连接流 | 删 | 是 | 仅允许删除已失效连接流 |
| `/flows/{flowId}/versions` | GET | 查询可部署版本 | 查 | 是 | 只返回已发布版本或由前端筛选已发布版本 |

## 9 实施顺序

1. 扩展连接流状态枚举，新增已失效，并将未部署展示文案调整为待部署。
2. 扩展表格列，补充连接流 ID、创建者、更新人。

## 11 系统级非功能设计

### 11.1 FMEA 影响分析

| 风险 | 影响 | 措施 |
|---|---|---|
| 无可部署版本 | 用户无法完成部署 | 提示暂无已发布版本，禁止确认部署 |
| 菜单状态错误 | 用户看到不应出现的操作 | 前端按状态控制菜单，后端继续校验状态 |
| 删除误操作 | 数据丢失 | 仅已失效可删除，并增加二次确认 |
| 停止误操作 | 运行中连接流被误停 | 停止前二次确认 |
| 复制 ID 失败 | 运维定位受影响 | 提示失败，允许用户手动复制 |
| 接口失败 | 列表状态与后端不一致 | 保留当前页面状态，提示失败原因，成功后刷新列表 |
| 未知状态 | 展示和操作不确定 | 状态展示默认占位，隐藏高风险操作 |

### 11.2 安全影响分析

部署、启动、停止、失效、恢复、删除依赖后端鉴权。前端按状态控制操作入口，后端仍需校验当前连接流状态和用户权限。删除入口仅在已失效状态展示，且删除前需要二次确认。

### 11.3 兼容性

历史“未部署”状态在前端统一展示为“待部署”。如果后续新增状态，可通过状态枚举和更多菜单映射扩展。旧数据缺少创建者、更新人时展示默认占位。

### 11.4 可运维

复制 ID 便于日志排查和接口联调；部署、状态变更和删除操作成功后刷新列表；失败时展示明确错误提示，便于用户判断是否需要重试或联系管理员。

### 11.5 资料

需要更新连接流列表操作说明，补充状态含义、更多菜单规则、部署版本选择、复制流、复制 ID 和危险操作确认说明。
