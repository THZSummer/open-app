## 1 需求价值和概述

当前左侧导航缺少运行记录入口，也缺少连接流运行记录列表和详情能力。用户无法集中查看连接流执行结果、触发参数、各节点入参、出参、耗时、成功与否和失败错误信息。本次新增运行管理页面，用于只读查看和排障。

价值：集中查询连接流执行结果；支持按条件筛选；展示单次执行链路；提升问题定位效率。

## 2 上下文分析

| 角色 | 关注点 |
|---|---|
| 运维人员 | 查看执行状态、耗时和错误原因 |
| 开发人员 | 根据触发参数、节点入参、出参、耗时和错误信息定位配置问题 |
| 平台管理员 | 查看连接流运行情况 |

运行管理与连接流、连接流版本关联，但不提供重跑、编辑、修改版本等写操作。

## 6 功能设计

### 6.1 业界方案实现

运行记录页面通常采用列表加详情抽屉模式，列表用于快速定位，详情用于展示执行链路，并按节点展示入参、出参、耗时、执行状态和失败错误上下文。

### 6.2 功能实现整体设计方案

运行管理页面由筛选区、操作区、表格区和详情抽屉组成。页面只读，不提供重跑、编辑连接流、修改版本等写操作。

### 6.3 功能实现

| 模块 | 设计 |
|---|---|
| 导航入口 | 左侧新增运行记录菜单 |
| 路由 | `/connect/run-log` |
| 筛选区 | 连接流名称、执行状态、触发方式、时间范围 |
| 表格区 | 执行 ID、连接流名称、版本、触发方式、状态、耗时、开始/结束时间 |
| 详情抽屉 | 基本信息、触发参数、节点执行明细、错误信息 |
| 节点执行明细 | 按执行顺序展示节点名称、状态、耗时、入参、出参，失败时展示错误信息 |
| JSON 展示 | 格式化展示触发参数、节点入参、节点出参和错误上下文 |

#### 6.3.1 接口设计

| URL | method | 功能 | 增删改查 | 鉴权 |
|---|---|---|---|---|
| `/flow-run-logs` | GET | 查询运行记录列表 | 查 | 是 |
| `/flow-run-logs/{executeId}` | GET | 查询运行详情 | 查 | 是 |

#### 6.3.2 数据模型设计

| 字段 | 类型 | 说明 |
|---|---|---|
| executeId | string | 执行 ID |
| flowId | string | 连接流 ID |
| flowName | string | 连接流名称 |
| versionNo | string | 连接流版本 |
| triggerType | string | 触发方式 |
| executeStatus | string | 执行状态 |
| duration | number | 整体执行耗时 |
| startTime/endTime | string | 开始、结束时间 |
| nodeLogs | array | 节点执行明细，包含节点状态、入参、出参、耗时和错误信息 |

## 7 系统级非功能设计

### 7.1 FMEA 影响分析

| 风险 | 影响 | 措施 |
|---|---|---|
| 列表查询失败 | 无法查看记录 | 展示错误并允许刷新 |
| 详情加载失败 | 无法排障 | 保留列表并提示失败 |
| JSON 过大 | 页面卡顿 | 抽屉分区展示，必要时折叠 |
| 字段缺失 | 展示异常 | 使用占位符 |

### 7.2 安全影响分析

运行记录可能包含请求参数和响应数据，页面访问依赖后端鉴权；前端不提供写操作。

### 7.3 兼容性

字段缺失时展示占位；后续新增触发方式和状态通过枚举扩展。

### 7.4 可运维

提供刷新、筛选、详情和节点级日志，便于排障。

### 7.5 资料

新增运行记录使用说明和字段说明。
