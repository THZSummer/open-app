## 一、项目概述

### 1.1 项目背景

本项目旨在为管理后台新增两个核心功能模块：**连接器管理**和**连接流管理**。这两个模块将支持企业内部系统对接以及平台API能力的封装与编排，实现业务流程自动化。

### 1.2 项目目标

- 提供完整的连接器生命周期管理（创建、配置、查询、删除）
- 提供可视化的连接流编排能力，降低使用门槛
- 支持多种触发类型（事件驱动、定时、手动）
- 参考钉钉连接平台的设计理念，提供友好的用户体验

### 1.3 核心概念

#### 连接器（Connector）

连接器是封装特定业务系统能力的组件，包含：

- **基本信息**：名称、描述、图标、状态等
- **触发事件（Trigger）**：外部系统状态变化时启动流程的事件
- **执行动作（Action）**：连接器暴露的可执行操作

#### 连接流（Flow）

连接流是通过图形化界面编排的自动化业务流程，包含：

- **流程类型**：
  - 业务流：事件驱动的流程
  - 定时流：定时触发的流程
  - 子流程：可被调用的公共流程
- **节点**：触发器、执行动作、条件分支、分支（if-else）、延时等
- **连线**：节点之间的数据流转关系

## 二、功能需求

### 2.1 连接器管理模块

#### 2.1.1 功能列表

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 连接器列表 | 展示所有连接器，支持分页、搜索、筛选 | P0 |
| 创建连接器 | 新建连接器，配置基本信息 | P0 |
| 编辑连接器 | 修改连接器的基本信息和配置 | P0 |
| 删除连接器 | 删除连接器（需确认） | P0 |
| 触发事件管理 | 配置连接器的触发事件列表 | P0 |
| 执行动作管理 | 配置连接器的执行动作列表 | P0 |

#### 2.1.2 详细功能说明

**连接器列表页面**

- 表格展示：连接器名称、描述、状态、触发事件数量、执行动作数量、创建时间
- 支持按名称关键词搜索
- 支持按状态筛选（全部/启用/禁用）
- 支持分页（每页10/20/50条）
- 操作列：编辑、删除

**创建/编辑连接器（弹窗表单）**

- 基本信息：
  - 连接器名称（必填，最大50字符）
  - 连接器描述（选填，最大500字符）
  - 图标（选填，支持上传或选择预设图标）
  - 状态（启用/禁用，默认启用）

- 触发事件列表（Tab页）：
  - 事件名称（必填）
  - 事件描述（选填）
  - 触发类型（Webhook / API轮询 / 定时）
  - 事件参数定义（JSON Schema格式）
  - 添加/编辑/删除事件

- 执行动作列表（Tab页）：
  - 动作名称（必填）
  - 动作描述（选填）
  - API配置（请求方法、URL、参数映射）
  - 响应参数定义（JSON Schema格式）
  - 添加/编辑/删除动作

### 2.2 连接流管理模块

#### 2.2.1 功能列表

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 连接流列表 | 展示所有连接流，支持分页、搜索、筛选 | P0 |
| 创建连接流 | 新建连接流，进入全屏流程画布 | P0 |
| 编辑连接流 | 修改连接流的配置和流程编排 | P0 |
| 删除连接流 | 删除连接流（需确认） | P0 |
| 流程编排画布 | 全屏可视化流程编辑器 | P0 |
| 节点拖拽 | 从节点库拖拽节点到画布 | P0 |
| 节点配置 | 配置选中节点的属性 | P0 |
| 连线配置 | 连接节点建立数据流转关系 | P0 |
| 流程保存 | 保存流程配置 | P0 |
| 并行执行支持 | 支持并行执行节点 | P0 |
| 循环执行支持 | 支持循环执行节点 | P0 |

#### 2.2.2 详细功能说明

**连接流列表页面**

- 表格展示：流程名称、类型、状态、节点数量、更新时间
- 支持按名称关键词搜索
- 支持按类型筛选（全部/业务流/定时流/子流程）
- 支持分页（每页10/20/50条）
- 操作列：编辑、删除
- 可在列表页查看流程图的缩略图

**创建/编辑连接流（弹窗表单）**

- 基本信息：
  - 流程名称（必填，最大50字符）
  - 流程描述（选填，最大500字符）
  - 流程类型（业务流 / 定时流 / 子流程）
  - 状态（草稿 / 已发布）

**全屏流程编排画布（使用@xyflow/react）**

- 布局：左侧节点库面板 + 中央流程画布 + 右侧属性配置面板
- 节点类型支持：
  - **触发器节点**：起始节点，定义流程如何被触发
  - **执行动作节点**：调用连接器的执行动作
  - **条件分支节点**：if-else逻辑判断
  - **延时节点**：等待指定时间后继续
  - **并行执行节点**：同时执行多个分支
  - **循环执行节点**：循环执行指定次数或条件满足前重复执行
- 画布功能：
  - 节点拖拽到画布
  - 节点选中高亮
  - 节点移动位置
  - 节点删除（选中后按Delete键或点击删除按钮）
  - 连线（从一个节点的输出拖拽到另一个节点的输入）
  - 删除连线（选中连线后按Delete键）
  - 画布拖拽（鼠标中键或空格+拖拽）
  - 画布缩放（使用@xyflow/react的Controls组件，支持缩放和重置大小）
  - 保存流程
  - 关闭画布（返回列表）

## 四、数据模型

### 4.1 连接器（Connector）

```javascript
{
  id: string,                    // 唯一标识
  name: string,                  // 连接器名称
  description: string,           // 连接器描述
  icon: string,                  // 图标URL
  status: number,                 // 状态：0-禁用，1-启用
  triggers: [                     // 触发事件列表
    {
      id: string,
      name: string,
      description: string,
      type: string,              // 触发类型：webhook/api/polling/schedule
      config: object,             // 触发器配置
      paramsSchema: object        // 参数Schema
    }
  ],
  actions: [                      // 执行动作列表
    {
      id: string,
      name: string,
      description: string,
      connectorId: string,
      method: string,             // HTTP方法
      endpoint: string,           // API端点
      requestSchema: object,      // 请求参数Schema
      responseSchema: object      // 响应参数Schema
    }
  ],
  createdAt: string,
  updatedAt: string
}
```

### 4.2 连接流（Flow）

```javascript
{
  id: string,
  name: string,
  description: string,
  type: string,                  // 流程类型：business/schedule/subflow
  status: number,                // 状态：0-草稿，1-已发布
  nodes: [                       // 节点列表
    {
      id: string,
      type: string,              // 节点类型：trigger/action/condition/delay
      position: { x: number, y: number },
      data: {
        label: string,
        config: object           // 节点配置
      }
    }
  ],
  edges: [                       // 连线列表
    {
      id: string,
      source: string,             // 源节点ID
      target: string,             // 目标节点ID
      sourceHandle: string,        // 源节点端口
      targetHandle: string,        // 目标节点端口
      label: string
    }
  ],
  createdAt: string,
  updatedAt: string
}
```

### 4.3 节点类型定义

```javascript
// 触发器节点配置
{
  triggerType: string,            // 触发类型
  connectorId: string,           // 关联的连接器ID
  triggerId: string,             // 触发事件ID
  cronExpression: string         // 仅定时触发时使用
}

// 执行动作节点配置
{
  connectorId: string,           // 关联的连接器ID
  actionId: string,              // 执行动作ID
  inputMapping: object,           // 输入参数映射
  outputMapping: object          // 输出参数映射
}

// 条件分支节点配置
{
  conditions: [
    {
      field: string,
      operator: string,          // eq/neq/gt/lt/gte/lte/contains
      value: string
    }
  ],
  defaultBranch: string          // 默认分支ID
}

// 延时节点配置
{
  delayType: string,             // fixed/dynamic
  duration: number,             // 延时时长（秒）
  durationField: string          // 仅动态延时时使用
}

// 并行执行节点配置
{
  branches: [                    // 并行分支列表
    {
      id: string,
      name: string,
      nodes: [                  // 分支内的节点
        { id: string, type: string, ... }
      ]
    }
  ],
  waitForAll: boolean            // 是否等待所有分支完成
}

// 循环执行节点配置
{
  loopType: string,              // times/while/until
  maxIterations: number,         // 最大迭代次数（仅times类型）
  condition: object,            // 循环条件（while/until类型）
  body: [                       // 循环体节点
    { id: string, type: string, ... }
  ]
}
```

### 6.5 组件交互设计

**节点拖拽流程**：
1. 从左侧节点库拖拽节点
2. 拖拽到画布上释放
3. 节点自动放置在释放位置
4. 节点自动选中，右侧属性面板显示配置选项
5. 用户配置节点属性
6. 点击画布空白处取消选中

**节点连线流程**：
1. 鼠标悬停在节点边缘，显示连接点
2. 从源节点拖拽连接点到目标节点
3. 释放后自动创建连线
4. 点击连线可选中，右侧面板可配置连线属性

**节点属性配置**：
1. 点击节点选中
2. 右侧面板显示该节点的配置选项
3. 根据节点类型显示不同的配置表单
4. 配置变更自动保存到流程数据
5. 点击画布空白处取消选中
