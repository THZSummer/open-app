# 综合测试数据生成脚本使用说明

## 脚本信息

- **文件路径**: `open-server/test/scripts/test-create-demo-data.sh`
- **文件大小**: 19KB
- **代码行数**: 548行
- **创建日期**: 2026-04-24

## 功能概述

该脚本用于生成综合测试数据，包括：
1. 创建分类数据（3个）
2. 创建API注册数据（8个）
3. 创建事件注册数据（7个）
4. 创建回调注册数据（4个）
5. 验证审批流程

## 使用方法

### 前置条件

1. 确保 `open-server` 服务已启动
   ```bash
   cd open-server
   mvn spring-boot:run
   ```

2. 确保服务运行在默认端口 `18080`

### 执行脚本

```bash
cd open-server/test/scripts
./test-create-demo-data.sh
```

## 特性说明

1. **唯一性保证**: 使用时间戳和随机数确保数据唯一性
2. **完整配置**: 每个资源都包含完整的属性配置
3. **审批配置**: 至少3个API/事件/回调配置了审批流程
4. **错误处理**: 对每个请求进行响应检查
5. **清晰输出**: 使用彩色输出和图标标识状态
6. **统计报告**: 自动生成创建统计和ID列表

## 注意事项

1. 脚本执行前请确保服务已启动
2. 如果数据库中已有相同数据，可能会创建失败（409冲突）
3. 可以多次执行脚本，每次会生成不同的测试数据
4. 脚本会保存创建的资源ID，方便后续操作
5. 每个请求之间有200ms延迟，避免请求过快

## 故障排查

### 服务未启动
```
✗ 分类创建失败
{"code":"000","messageZh":"服务未响应"}
```
**解决方案**: 确保 open-server 服务已启动

### 端口冲突
```
curl: (7) Failed to connect to localhost port 18080
```
**解决方案**: 检查服务端口配置，或修改脚本中的 BASE_URL

## 脚本维护

如需修改测试数据，可以编辑脚本中的数据数组：

- 分类数据: `create_categories()` 函数中的 `categories` 数组
- API数据: `create_apis()` 函数中的 `apis` 数组
- 事件数据: `create_events()` 函数中的 `events` 数组
- 回调数据: `create_callbacks()` 函数中的 `callbacks` 数组
